Tato stránka zobrazuje základní informace týkající se clusteru high-availability a členských serverů. Dále tato stránka nabízí nástroje určené k vytváření, odstraňování nebo údržbě clusteru high-availability a členských serverů.

Stav clusteru high-availability

Toto pole zobrazuje celkový stav clusteru high-availability a upozorňuje na veškeré chyby. Následující tabulka představuje všechny možné podmínky zobrazené v tomto poli:

Stav Popis
Samostatný Servery nelze propojit a nelze vytvořit cluster high-availability.
V pořádku Celkový stav clusteru high-availability je dobrý. Replikace dat z aktivního serveru na pasivní server funguje standardně.
Příprava Servery byly úspěšně propojeny v rámci clusteru high-availability, replikace dat však ještě nebyla dokončena. Přepnutí nelze nyní aktivovat. Další informace získáte v části Přepnutí.
Načítání Systém načítá stav clusteru. Přepnutí lze provést až po dokončení načítání stavu.
Upozornění Cluster HA není v pořádku, služby však fungují standardně.
Výstraha Aktivní ani pasivní server nefunguje správně. Cluster high-availability nemůže správně fungovat. Zkontrolujte stav veškeré úložné kapacity (např. skupin RAID Group, skupin disků, svazků a jednotek iSCSI LUN).

Následující tabulka obsahuje zprávy upozornění, které se mohou zobrazit v případě chyby systému. Pokud se zobrazí více než jedna zpráva, můžete se v rámci těchto zpráv pohybovat tlačítky Dopředu/Zpět.

Upozornění / Chyba Popis
Chyba sledované služby U jedné nebo více sledovaných služeb došlo k chybě.
Omezený režim úložného místa Došlo k omezenému režimu úložného místa (např. skupiny RAID Group, skupiny disků, svazku, jednotky iSCSI LUN a mezipaměti SSD) na aktivním nebo pasivním serveru. Zkontrolujte stránku Stav úložiště.
Poškození úložného místa Došlo k poruše úložného místa (např. skupiny RAID Group, skupiny disků, svazku, jednotky iSCSI LUN a mezipaměti SSD) na aktivním nebo pasivním serveru. Zkontrolujte stránku Stav úložiště.
Nepodařilo se ověřit pasivní server K této chybě může dojít tehdy, pokud je pasivní server v režimu offline anebo není možné připojit aktivní server k pasivnímu serveru pomocí připojení prezenčního signálu. V tomto případě nebude funkce přepnutí k dispozici.
Nelze provést replikaci dat na pasivní server K této chybě může dojít z důvodu selhání svazku úložiště nebo disku. V tomto případě nebude funkce přepnutí k dispozici. Zkontrolujte stránku Stav úložiště.
Pasivní server nebyl vybrán Pokud dojde k odpojení pasivního serveru z clusteru high-availability, bude zobrazen tento stav. V tomto případě nebude funkce přepnutí k dispozici. Chcete-li pasivní server znovu připojit, klikněte na rozevírací nabídku Údržba a poté klikněte na položku Připojit. Pokud pasivní server opět připojíte, bude vyžadována kompletní replikace dat.

Informace o clusteru high-availability

Toto pole se nachází pod informací o stavu clusteru high-availability a zobrazuje základní informace o uživateli. Pokud cluster high-availability neexistuje, obsahuje toto pole vysvětlivky k funkcím clusterů high-availability.

Toto pole obsahuje následující informace:

  • Název serveru clusteru: Název serveru používaného clusterem high-availability.
  • IP adresa clusteru: IP adresa používaná clusterem high-availability. Bez ohledu na to, na jakém serveru jsou služby spuštěny, se mohou uživatelé připojit ke clusteru high-availability pomocí této IP adresy.
  • Čas sestavení: Datum vytvoření clusteru high-availability.

Ovládací tlačítka clusteru high-availability

Pod informacemi o clusteru high-availability se nachází následující dvě tlačítka:

  1. Vytvoření clusteru HA: Pokud neexistuje žádný cluster high-availability, můžete kliknutím na toto tlačítko propojit oba servery a vytvořit cluster high-availability. Další informace o vytváření clusteru high-availability získáte v části Průvodce Synology High Availability.
  2. Správa: Toto tlačítko je k dispozici pouze tehdy, pokud již cluster high-availability existuje. Obsahuje následující možnosti:
    • Průvodce správou: Spustí průvodce, který obsahuje pokyny pro různé procesy správy.
    • Přepnutí: Zahájí proces přepnutí z aktivního serveru na pasivní server. Během tohoto procesu pozastaví aktivní server zajišťování služeb a bude plnit funkci nového pasivního serveru. Poté bude původní pasivní server plnit funkci nového aktivního serveru a zahájí provoz všech služeb clusteru high-availability. Tato funkce není k dispozici, pokud pasivní server vykazuje chybu nebo nebyl vybrán.
    • Aktualizace firmwaru DSM: Na stránce Ovládací panel otevřete možnost Aktualizace firmwaru DSM. V této části lze aktualizovat oba servery.
    • Odebrat cluster HA: Odebere cluster high-availability.
    • Odpojit pasivní server: Odpojte pasivní server od clusteru. Po přidání serveru ke clusteru high-availability po předchozím odpojení bude vyžadována úplná replikace dat. Proto tuto možnost doporučujeme, pokud chcete existující pasivní server nahradit novým serverem.
    • Připojit pasivní server: Přidá ke clusteru pasivní server. Tato možnost je k dispozici pouze tehdy, pokud není součástí clusteru high-availability žádný pasivní server.
    • Vypnout cluster HA: Vypne aktivní i pasivní server.
    • Vypnout aktivní server: Vypne aktivní server.
    • Vypnout pasivní server: Vypne pasivní server.
    • Restartovat pasivní server: Restartuje pasivní server.
    • Deaktivovat zvukový signál pasivního serveru: Pozastaví zvukový signál pasivního serveru, pokud dojde k chybě. Dojde-li k jiné chybě, pasivní server i přesto vydá zvukový signál.

Poznámka:

  • Po odebrání clusteru high-availability bude v případě vytvoření nového clusteru vyžadována kompletní replikace dat.
  • Pokud vypnete nebo odpojíte pasivní server, služby nelze přepnout, pokud dojde k chybě aktivního serveru.

Stav replikace serveru a dat

Tato část obsahuje obrázky zobrazující stav aktivních a pasivních serverů a dále také tok replikace dat mezi servery.

Obrázek replikace dat označuje následující možné podmínky:

  • Replikace dat nefunguje správně: Tento stav je označen tokem dat směrem z aktivního serveru na pasivní server.
  • Bez replikace dat: Tento stav je označen chybějícím zobrazením toku dat.

Informace o serveru

Tato část se nachází pod obrázky serverů. V této části jsou zobrazeny informace o aktivních a pasivních serverech. Pokud nebude vybrán žádný server, bude tato část obsahovat hodnotu Není k dispozici.

  • Název serveru: Název serveru Synology.
  • Název modelu: Model serveru Synology.
  • Sériové číslo: Sériové číslo serveru Synology.
  • Stav ventilátoru: Stav chladících ventilátorů serveru. Pokud ventilátor nefunguje, bude zobrazeno upozornění. V tomto případě se obraťte na svého místního distributora.
  • Teplota: Teplota serveru Synology.
  • Stav napájení: Označuje, zda všechny napájecí zdroje správně fungují. Pokud je model serveru vybaven redundantními napájecími zdroji, zobrazí se v případě nesprávného fungování upozornění.
