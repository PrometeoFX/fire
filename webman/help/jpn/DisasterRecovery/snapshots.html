スナップショット

このページでは、スナップショットを撮影、削除、編集したり、スナップショットの予定を設定したり、維持設定を行ったりすることができます。

各共有フォルダまたは iSCSI LUN 毎に、一度に最大 256 個のスナップショット コピーを保管できます。ナップショットは、ファイル全体ではなくブロック毎に維持されるため、ディスクの使用量を最小化します。共有フォルダのファイルや iSCSI LUN のブロックを更新したり、削除したりすると、多くのスペースが使用されます。

共有フォルダ

共有フォルダのスナップショットを撮る：

  1. スナップショットを撮影したい共有フォルダを選択します。
  2. [スナップショットを撮る] をクリックします。
  3. 撮影するスナップショットの説明を [説明] に入力してください。
  4. [ロック] オプションで、自動削除が実行されるとき、スナップショットが維持設定を無視するかどうかを定義します。
  5. ［OK］ をクリックすると、スナップショットが撮影されます。

注：

  • [コントロール パネル] > [共有フォルダ] > [編集] ページの設定の中で、Windows ACL 権限だけがスナップショットに記録されます。

共有フォルダのスナップショットを参照する：

  1. 参照したいスナップショットの共有フォルダを選択します。
  2. [スナップショット リスト]をクリックします。
  3. 参照したいスナップショットを選択して [参照] をクリックします。

注：

  • 暗号化された共有フォルダについては、スナップショットを参照することはできません。
  • スナップショットを参照できない場合は、[コントロール パネル] > [共有フォルダ] > [編集] を選択して [スナップショットを検出できるようにする] を有効にしてください。

共有フォルダのスナップショットを削除する：

  1. 削除したいスナップショットの共有フォルダを選択します。
  2. [スナップショット リスト]をクリックします。
  3. 削除したいスナップショットを選択し、[削除] をクリックします。（Shift または Ctrl を押しながら選択すると、複数選択ができます。）
  4. 確認メッセージが表示されます。削除を確定するには [削除] をクリックします。

共有フォルダのスナップショットを編集する：

  1. 編集したいスナップショットの共有フォルダを選択します。
  2. [スナップショット リスト]をクリックします。
  3. 編集したいスナップショットを選択して[参照]をクリックします。
  4. 自動削除が行われたときに、スナップショットに維持設定を無視させるかどうかは、スナップショットの説明を [説明] で編集し、[ロック] オプションを使って決定します。
  5. [OK] をクリックして、設定を保存します。

予約スナップショットを構成する：

スナップショットの撮影を予約しておくと、データ保護マネージャが指定された時間と頻度で自動的にスナップショットを撮影してくれるため、手動でスナップショットを撮る手間を省くことができます。

  1. 共有フォルダを選択します。
  2. [スケジュールの設定] をクリックします。
  3. [スケジュール] タブで [スナップショットの予定を有効にする] にチェックマークを付けて、スナップショットを撮影する時間と頻度を設定します。この共有フォルダのスナップショットの予定をキャンセルするには、チェックボックスのチェックマークを外してください。
  4. [OK] をクリックして、設定を保存します。
  5. 確認メッセージが表示されます。[OK] をクリックして設定を確定します。

注：

  • 予約したスナップショットの構成は、[コントロール パネル] > [更新と復元] > [設定のバックアップ] 機能には含まれません。

予約スナップショットの維持設定を行う：

  1. 共有フォルダを選択します。
  2. [スケジュールの設定] をクリックします。
  3. [維持] タブを選択します。
  4. 維持設定を行います。以下のいずれかのオプションを選択してください。
    • 上限に達したら古いスナップショットを削除する：スナップショットが最大数に達したら、古いスナップショットから新しいもので上書きされていきます。
    • 常にスナップショットを維持する：スナップショットが最大数に達したら、データ保護マネージャが既に撮影したものを削除せずに、新しいスナップショットの撮影を中止します。
    • 次のポリシーに基づいてスナップショットを維持する: 指定した時間枠で維持規則を設定できます。詳しくは、維持ポリシーを参照してください。
  5. [OK] をクリックして、設定を保存します。

iSCSI LUN

iSCSI LUN のスナップショットを撮る：

  1. スナップショットを撮影したい iSCSI LUN を選択します。
  2. [スナップショットを撮る] をクリックします。
  3. 撮影するスナップショットの説明を [説明] に入力してください。
  4. 自動削除が行われたときに維持設定を無視するかどうかについて、[ロック] ドロップダウン リストから [はい/いいえ] を選択します。
  5. ［OK］ をクリックすると、スナップショットが撮影されます。

iSCSI LUN のスナップショットを削除する：

  1. 削除したいスナップショットの iSCSI LUN を選択します。
  2. [スナップショット リスト]をクリックします。
  3. 削除したいスナップショットを選択し、[削除] をクリックします。（Shift または Ctrl を押しながら選択すると、複数選択ができます。）
  4. 確認メッセージが表示されます。[はい] をクリックして削除を確認します。

iSCSI LUN のスナップショットを編集する：

  1. 編集したいスナップショットの iSCSI LUN を選択します。
  2. [スナップショット リスト]をクリックします。
  3. 編集したいスナップショットを選択して[参照]をクリックします。
  4. スナップショットを編集するには、次のような方法があります。
    • [説明] でスナップショットの説明を編集します。
    • アプリケーション コンシステントなスナップショットが有効になっているときには、コンシステント状態を選択します。
    • 自動削除が行われたときに維持設定を無視するかどうかについて、[ロック] ドロップダウン リストから [はい/いいえ] を選択します。
  5. [OK] をクリックして、設定を保存します。

予約スナップショットを構成する：

スナップショットの撮影を予約しておくと、データ保護マネージャが指定された時間と頻度で自動的にスナップショットを撮影してくれるため、手動でスナップショットを撮る手間を省くことができます。

  1. iSCSI LUN を選択します。
  2. [スケジュールの設定] をクリックします。
  3. [スケジュール] タブで [スナップショットの予定を有効にする] にチェックマークを付けて、スナップショットを撮影する時間と頻度を設定します。この共有フォルダのスナップショットの予定をキャンセルするには、チェックボックスのチェックマークを外してください。
  4. [OK] をクリックして、設定を保存します。

予約スナップショットの維持設定を行う：

  1. iSCSI LUN を選択します。
  2. [スケジュールの設定] をクリックします。
  3. [維持] タブを選択します。
  4. 維持設定を行います。以下のいずれかのオプションを選択してください。
    • 上限に達したら古いスナップショットを削除する：スナップショットが最大数に達したら、古いスナップショットから新しいもので上書きされていきます。
    • 常にスナップショットを維持する：スナップショットが最大数に達したら、データ保護マネージャが既に撮影したものを削除せずに、新しいスナップショットの撮影を中止します。
    • 次のポリシーに基づいてスナップショットを維持する: 指定した時間枠で維持規則を設定できます。詳しくは、維持ポリシーを参照してください。
  5. [OK] をクリックして、設定を保存します。
  6. 確認メッセージが表示されます。[OK] をクリックして確定します。

iSCSI LUN スナップショットのコンシステント状態を構成する：

  1. iSCSI LUN を選択します。
  2. [スケジュールの設定] をクリックします。
  3. アプリケーション タブを選択します。
  4. コンシステント状態を切り換えるには、[アプリケーション コンシステントなスナップショットを有効にする] にチェックマークを付けます。
    • アプリケーション コンシステント：このコンシステント状態のスナップショットは、DSM とコミュニケートする操作環境によって、Windows サーバーまたは VMware vCenter サーバーに Synology Snapshot Manager がインストールされている場合に限って取得されます。DSM でアプリケーション コンシステントなスナップショットが取得されると、アプリケーションに通知が送信され、データの統合性を図るために、メモリからスナップショットが取得された LUN へすべてのデータを転送します。

      注：

      • 操作環境に Synology Snapshot Manager がインストールされている場合は、チェックマークを付けてもアプリケーション コンシステントなスナップショットは無効のままになります。
    • クラッシュ コンシステント：クライアントに Synology Snapshot Manager がインストールされていなければ、LUN のクラッシュ コンシステントなスナップショットしか取得されません。クラッシュ コンシステントなスナップショットが撮影されると、メモリから LUN にデータをフラッシュするためにアプリケーションには通知が送信されません。したがって、データの整合性も保証されません。
  5. [OK] をクリックして、設定を保存します。

維持ポリシー

このオプションは、ボリュームのスペースを節約するためにスナップショットのバージョンが占有する最大容量を指定するのに理想的な方法ですが、長期間にわたってスナップショットを維持する必要があるかもしれません。

Synology は GFS(｢Grandfather-Father-Son｣親子三代）維持ポリシーを採用しています。スナップショット バージョンが占有する最大容量を維持しておく期間を次のように設定することができます。毎時、毎日、毎週、毎月、毎年。

データ保護マネージャは、それぞれの時間枠において設定された最大数のスナップショットを維持します。特定の時間枠に複数のスナップがある場合は、最も古いものだけが維持されます。例えば、週間スナップショットを10個に設定した場合、過去10週間分は最も古いスナップショットが維持されます（1週間に複数のスナップショットが撮影された場合）。

さらに、最近撮影したスナップショット バージョンから簡単に復元できるように、デフォルトでは最新のスナップショットが撮影されてから1時間以内に撮影したすべてのスナップショット バージョンが維持されます。維持規則により自動的にスナップショットが削除されないように、スナップショットをロックすることも可能です。
