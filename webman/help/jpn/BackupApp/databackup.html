データ バックアップ

[バックアップと複製] の [バックアップ] タブで、DiskStation の特定のデータを手動、または指定した時間にバックアップすることができます。この機能を使うと、システム設定、共有フォルダの内容、パッケージ データがローカル、ネットワーク、またはクラウドのバックアップ先にバックアップされます。

バックアップタスクを作成する：

  1. [作成] > [データ バックアップ タスク] を選択します。
  2. ウィザードに従って設定を終了します。詳細は、バックアップ先をお読みください。

バックアップ タスクを管理する：

既存のタスクを選択し、次のボタンの 1 つをクリックします。

  • 編集：選択したタスクのバックアップ先、バックアップ元、スケジュール、その他の設定を編集できます。
  • 削除：選択したタスクを削除します。
  • キャンセル：選択した実行中のタスクをキャンセルします。
  • 今すぐバックアップ：選択したタスクを即座に実行します。

バックアップ ローテーション

バックアップ ローテーションは、バージョン管理メカニズムです。古い、不要なバージョンを自動的に削除して、ストレージ スペースを節約することで、バックアップ バージョンを管理します。この機能は [タスク設定] で有効にできます。または、バックアップ タスクを作成するときに設定することも可能です。

バージョンを交代させる：

  1. バージョン ローテーションを適用するバックアップ タスクを選択し、[編集] をクリックします。
  2. [タスク設定] タブを開き、[バックアップ ローテーションを有効にする] にチェックマークを付けます。
  3. ローテーション方式を選択してください。
    • 古いバージョンから：基本的なバージョン ローテーション方式です。ローテーションの条件が揃ったら、古いバージョンから削除されていきます。
    • Smart Recycle：インテリジェントなバージョン ローテーション方式です。ローテーションの条件が揃ったら、ローテーション メカニズムが選択したバージョンを削除します。ローテーション メカニズムは次のとおりです。
      • 過去 24 時間分の時間ごとのバージョンを維持する：システムは、毎時作成される最も古いバージョンを維持します。
      • 日時バージョンを過去 1 日～1 ヶ月分維持する：システムは、毎日作成される最も古いバージョンを維持します。
      • 週次バージョンを 1 ヶ月分維持する：システムは、毎週作成される最も古いバージョンを維持します。
  4. 最大バージョン数を指定してください。バージョン数がこの数を超えると、バージョン ローテーションが開始されます。この数には、ロックされたバージョンは含まれません。
  5. 反復バックアップ予約が設定されている場合は、バックアップ バージョンをどのように維持するかを予測したプレビューと、ローテーション設定に基づいた最も古い復元ポイント時刻が表示されます。  
  6. [OK] をクリックして、設定を保存します。

注意：

  • ロックしたバージョンは絶対に削除できません。詳細は、バックアップ先をお読みください。
  • マルチ バージョンのバックアップ タスクにのみバージョン ローテーション機能を適用できます。
  • バージョン ローテーションは、DSM 5.1 以下のバージョンではサポートされていません。
  • ローテーションは、設定が変更されるかバックアップ タスクが完了するまでは実行されません。
  • 最大バージョン数を計算する際は、ロックしたバージョンは除外されます。

一般的な注意：

  • タスクの名前には、1 から 32 個のユニコード文字を使用できます。ただし、次の文字は使用できません。
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • バックアップのフルパスの最大長は 2048 文字です。
  • AFP を介して 32000 個以上のファイルかフォルダを EXT3 共有フォルダにアップロードすると、共有フォルダを含むすべてのバックアップ タスクが失敗します。
  • 暗号化したフォルダは、マウントしても、マウントしなくてもバックアップできます。ただし、マウントされずに復元され、スタートアップ時には自動的にマウントされません。暗号化した共有フォルダの暗号化キーを入力するか、インポートしてマウントする必要があります。
  • 暗号化したフォルダを別の暗号化フォルダにバックアップすることはできません。
  • バックアップ ソースまたはバックアップ先としてリモート フォルダを使用することはできません。
  • ボリュームがバックアップ先のときには、タスクの所有者しかバックアップタスクを実行することができません。詳細は、バックアップ先をお読みください。
  • 特定アプリケーションのバックアップに対応しています。特定のパッケージがバックアップ、バックアップ コンテンツ、制限に対応しているかどうかを確認するには、各アプリケーションのヘルプを参照してください。
  • DiskStation は、書き込み可能な EXT3/EXT4 外部デバイスがないアプリケーションのバックアップには対応していません。
  • DiskStation は、書き込み可能な EXT3/EXT4 外部デバイスがないボリュームへのバックアップには対応していません。
  • DiskStation は、EXT3/EXT4 外部デバイスで作成した共有フォルダしかバックアップしません。
  • システムは書き込み可能なボリュームなしでは、アプリケーションをバックアップできません。

Windows ACL 設定のバックアップ：

バックアップ先のファイルシステムによっては、DiskStation 上のファイルの Windows ACL 設定もバックアップされる場合があります。

  • バックアップ先のファイルシステムに関係なく、「ローカル バックアップ先」、「ネットワーク バックアップ先」、「パブリッククラウドのバックアップ先」で Windows ACL 設定をバックアップできます。
  • バックアップ先のファイルシステムが EXT4/BTRFS の場合は、「共有フォルダ同期」で Windows ACL 設定をバックアップできます。
