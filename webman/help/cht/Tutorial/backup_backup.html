備份您的 eCore Server 以保障資料安全

概述

伺服器故障或儲存空間損毀等不可預期的意外對您的寶貴資料造成嚴重威脅。 eCore Server 提供使用者各種不同方案來備份重要資料，包含本地及遠端備份，時時為您做好準備，讓您無後顧之憂。 您可以依據企業的需求和情況，選擇一種或更多備份方案來降低資料遺失的風險。

1. 備份至外接儲存裝置

進行本地備份時，您可能會選擇將備份目的設為同一個儲存空間、同一組 RAID，或甚至是同一顆實體硬碟的另一個資料夾。 在這個情況下，如果您的資料遭到誤刪，就能從另一個資料夾的備份資料中復原。 然而當您不巧遇到伺服器故障或是磁碟損毀的狀況，您的資料和備份的副本都可能會遺失。 因此，建議您將重要的資料備份到外接的儲存裝置，而非相同的伺服器上。 eCore Server 讓使用者不僅可以將備份目的地設定在本機磁碟，也可以設定到外接的 USB 裝置。 本項功能可於主選單 > 備份 & 同步啟動。 詳細步驟請參閱這裡。

2. 備份至另一台 eCore Server

將您的資料備份至外接儲存裝置可以加強資料在有需要時能夠被利用的程度，但試想您設置伺服器和儲存裝置的地點有可能暴露於天災人禍，諸如火災、地震等風險中。 eCore Server 讓使用者可以建立異地備份，藉由將備份的資料透過網路傳送到置放於其他地點的另一台伺服器上，把起因於伺服器設置地點發生災害而造成資料遺失的風險降到最低。 您可以為備份的資料傳輸加密，以更高的安全性保護您的重要資料。 啟動段落分塊 (block-level) 備份選項，您的 NAS 將只會備份經修改的段落，這項功能在透過網際網路執行網路備份時特別實用。 本項功能可於主選單 > 備份 & 同步啟動。 詳細步驟請參閱這裡。

3. 備份至公有雲儲存空間

NAS 伺服器之間的備份方案需要兩台或更多的伺服器，數顆安裝於伺服器裡的硬碟，以及置放伺服器的地點。 您的企業的情況可能並不允許建立數台 NAS 裝置的異地備份。 為解決無法利用多台 NAS 進行網路備份的問題，Synology 提供另一種備份方案，支援數個公有雲服務，讓您能夠將資料直接備份到您的公有雲帳戶中。 本項功能可於主選單 > 備份 & 同步啟動，或可依照您慣用的公有雲儲存空間至套件中心啟動服務。 詳細步驟請參閱這裡。
