資料備份

在備份 & 同步的備份頁籤下，您可以手動備份 DiskStation 上的資料，或依自訂的排程執行備份。 透過此功能，您可將系統設定檔、共用資料夾內容及套件資料備份至本地、網路或雲端備份目的地。

若要新增備份任務：

  1. 選擇新增 > 資料備份任務。
  2. 依照設定精靈的指示完成設定。 請參閱備份目的地來瞭解更多資訊。

若要管理備份任務：

選擇現有的任務並按一下以下按鈕：

  • 編輯：修改所選任務的目的地、來源、排程及其它設定。
  • 刪除：刪除所選任務。
  • 取消：取消所選之正在執行的任務。
  • 立即備份：立即備份所選任務。

自動刪除備份

自動刪除備份版本為版本管理的機制。 它可有效的保管備份並避免您的儲存空間被過多的備份檔案佔據。 您可以在任務設定頁面或在建立備份任務時啟動此功能。

若要自動刪除版本：

  1. 選擇您想要套用自動刪除版本條件的被分任務並點一下編輯。
  2. 前往任務設定並勾選啟動自動刪除備份版本 。
  3. 您可以選擇以下任何一個方式設定自動刪除：
    • 從最早版本開始刪除為最基本的刪除版本方法。 當觸發刪除條件時，從時間最舊的版本開始刪除。
    • Smart Recycle 為一種智慧型的刪除版本方法。 當觸發刪除條件時，系統將刪除 Smart Recycle 選定的版本。 備份保留機制如下：
      • 過去 24 小時內，每小時保留一個版本： 系統會保留每小時最早建立的備份版本。
      • 過去 1 天至 1 個月內，每天保留一個版本： 系統會保留每天最早建立的備份版本。
      • 過去 1 個月之前，每週保留一個版本： 系統會保留每週最早建立的備份版本。
  4. 指定最高版本數量。 當版本數量超過該數字，則開始進行自動刪除。 此數字不包含鎖定版本數量。
  5. 若您有設定重複性備份排程，系統將會根據您的自動刪除設定顯示您的備份版本如何保存及最快復原時間的預覽。   
  6. 按一下確定來儲存設定。

注意：

  • 鎖定的版本不會被刪除。 請參閱備份目的地來瞭解更多資訊。
  • 只有多版本備份任務提供自動刪除備份版本功能。 
  • 備份目的地為 DSM 5.1 以下的系統無法使用自動刪除版本功能。
  • 備份任務設定改變後，自動刪除功能將立刻開始執行。
  • 系統計算最高版本數量時將不包含鎖定的版本。

一般注意事項：

  • 任務名稱必須設定為 1 到 32 個 Unicode 字元，但不可為下列字元：
    ！ " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • 備份的完整資料夾路徑，最大長度限制為 2048 個字元。
  • 如果您透過 AFP 將超過 32000 個檔案或資料夾上傳到 EXT3 共用資料夾，則所有備份任務 (包含該共用資料夾) 將會失敗。
  • 無論加密資料夾為卸載或裝載，皆可備份加密資料夾。但還原後會一律備份成卸載且不是開機時自動裝載的資料夾，您需要輸入或匯入該資料夾的加密金鑰才能裝載該資料夾。
  • 系統不支援將加密資料夾備份至另一個加密資料夾。
  • 系統不支援使用遠端資料夾作為備份來源或備份目的地。
  • 如果備份目的地是儲存空間，則只有任務擁有人可以執行立即備份。 請參閱備份目的地來瞭解更多資訊。
  • 系統支援備份特定的套件。 關於套件是否支援備份，以及備份的具體內容與限制，請參閱應用程式內各套件的說明。
  • 系統不支援 DiskStation 機種在沒有可寫入的 EXT3 或 EXT4 外接裝置的情況下備份應用程式。
  • 系統不支援 DiskStation 機種在沒有可寫入的 EXT3 或 EXT4 外接裝置的情況下備份至儲存空間。
  • DiskStation 機種僅能備份 EXT3 或 EXT4 外接裝置上的共用資料夾。
  • 系統不支援在沒有可寫入的儲存空間情況下備份應用程式。

備份 Windows ACL 設定：

DiskStation 上檔案的 Windows ACL 設定可能也會一併備份 (視目的地檔案系統而定)：

  • 本地備份目的地、網路備份目的地、公有雲備份目的地可以備份 Windows ACL 設定，不受限於目的地的檔案系統。
  • 當目的地的檔案系統是 EXT4/BTRFS 時，共用資料夾同步可以備份 Windows ACL 設定。
