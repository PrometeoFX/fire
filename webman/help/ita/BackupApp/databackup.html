Backup dei dati

Nella scheda Backup di Backup e Replica, è possibile eseguire il backup di dati specifici su DiskStation manualmente o secondo una pianificazione personalizzata. Con questa funzionalità, la configurazione di sistema, il contenuto delle cartelle condivise e dati pacchetto possono essere sottoposti a backup su destinazione di backup locale, di rete o cloud.

Per creare un'attività di backup:

  1. Scegliere Crea > Attività di backup dati.
  2. Seguire la procedura guidata per completare la configurazione. Visitare Destinazione Backup per maggiori informazioni.

Gestire attività di backup:

Selezionare un'attività esistente e fare clic su uno dei pulsanti seguenti:

  • Modifica: per modificare destinazione, origine, programmazione e altre impostazioni dell'attività selezionata.
  • Elimina: per eliminare l'attività selezionata.
  • Annulla: per annullare l'attività selezionata attualmente in esecuzione.
  • Backup ora: esegue immediatamente il backup dell'attività selezionata.

Rotazione backup

La rotazione backup è un sistema di gestione delle versioni. Contribuisce a gestire le versioni backup eliminando automaticamente le versioni più vecchie e non necessarie e liberando, potenzialmente, lo spazio di archiviazione. È possibile abilitare questa funzione su Impostazioni attività oppure configurarla quando si crea un'attività di backup.

Per ruotare le versioni:

  1. selezionare l'attività di backup a cui applicare la rotazione di versione e fare clic su Modifica.
  2. Andare alla scheda Impostazioni attività e selezionare Attiva rotazione backup.
  3. Qui sarà possibile scegliere uno dei due metodi di rotazione:
    • Dalla versione più recente:metodo di rotazione delle versioni di base. Quando viene attivata la rotazione, la versione più recente sarà rimossa per prima.
    • Smart Recycle:metodo di rotazione delle versioni intuitivo. Quando viene attivata la rotazione, il sistema rimuoverà le versioni selezionate dal meccanismo di mantenimento. Il meccanismo di mantenimento è il seguente:
      • Versioni orarie dalle 24 ore passate:il sistema mantiene la versione più recente creata ogni ora.
      • Versioni giornaliere da 1 giorno trascorso a 1 mese:il sistema mantiene la versione più recente creata ogni giorno.
      • Versione settimanale dopo 1 mese:il sistema mantiene la versione più recente creata ogni settimana.
  4. Specificare il numero di versioni massimo. Quando il numero di versioni supera questo numero, ha inizio la rotazione delle versioni. Il numero non include le versioni bloccate.
  5. Se è stato configurato un programma di backup ripetuto, sarà visualizzata un'anteprima della stima del metodo di mantenimento delle versioni di backup e il punto nel tempo del recupero più recente in base alle impostazioni di rotazione.   
  6. Fare clic su OK per salvare le impostazioni.

Note.

  • le versioni bloccate non saranno mai rimosse. Visitare Destinazione Backup per maggiori informazioni.
  • Solo le attività di backup archiviate forniscono le funzioni di rotazione delle versioni.
  • La rotazione versioni non è supportata in DSM 5.1 o inferiori.
  • La rotazione sarà attivata subito dopo la modifica dell'impostazione o al termine dell'attività di backup.
  • Le versioni bloccate sono escluse quando si calcola il numero massimo di versioni.

Note generali:

  • I nomi delle attività possono includere da 1 a 32 caratteri Unicode, ma non possono contenere i seguenti caratteri:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • La lunghezza massima del percorso completo della cartella per il backup è 2048 caratteri.
  • Se si caricano più di 32.000 file o cartelle via AFP in una cartella condivisa EXT3, tutte le attività di backup della cartella condivisa non verranno eseguite in modo corretto.
  • Una cartella crittografata può essere sottoposta a backup oppure montata o smontata. Tuttavia, verrà sempre ripristinata come smontata e non sarà montata automaticamente all'avvio. Verrà richiesto di inserire o importare la chiave di crittografia della cartella condivisa crittografata per montarla.
  • Il backup di una cartella crittografata a un'altra cartella crittografata non è supportato.
  • L'utilizzo di cartelle remote come sorgente di backup o destinazione di backup non è supportato.
  • Se la destinazione di backup è un volume, solo il proprietario dell'attività sarà in grado di eseguire l'attività di backup. Visitare Destinazione Backup per maggiori informazioni.
  • Il backup di determinate applicazioni è supportato. Per verificare se un pacchetto specifico supporta il backup, e il contenuto del backup e le limitazioni, fare riferimento alla guida di ciascuna applicazione.
  • DiskStation non supporta il backup di applicazioni senza un dispositivo esterno EXT3/EXT4 scrivibile.
  • DiskStation non supporta il backup in un volume senza un dispositivo esterno EXT3/EXT4 scrivibile.
  • In DiskStation è solo possibile eseguire il backup delle cartelle condivise create nei dispositivi esterni EXT3/EXT4.
  • Il sistema non è in grado di eseguire il backup delle applicazioni senza un volume scrivibile.

Backup delle impostazioni di Windows ACL:

può essere anche effettuato il backup dei file delle impostazioni di Windows ACL su DiskStation a seconda dei file system di destinazione:

  • Destinazione di backup locale, Destinazione di backup di rete e Destinazione di backup cloud pubblico possono essere sottoposte a backup delle impostazioni di Windows ACL senza tener conto del file system di destinazione
  • La Sincronizzazione cartelle condivise può eseguire il backup delle impostazioni Windows ACL quando il file system di destinazione è EXT4/BTRFS.
