Backup del dati sul eCore Server per garantire la sicurezza dei dati

Panoramica

Malfunzionamenti imprevisti del server o crash del volume costituiscono un rischio per i dati importanti. Il eCore Server offre agli utenti diverse alternative per eseguire sia localmente che da remoto copie di backup sicure dei dati più importanti, in modo da essere sempre preparati in caso di problemi. È possibile scegliere di eseguire il backup in o più modi per ridurre il rischio di perdita dei dati, in relazione ai requisiti e alle situazioni lavorative.

1. Backup su un dispositivo di archiviazione esterno

Eseguendo un backup locale, è possibile scegliere di impostare la destinazione del backup su un'altra cartella nello stesso volume, gruppo RAID o anche sullo stesso disco fisico. In questo caso, se i dati vengono inavvertitamente eliminati, sarà possibile recuperarli dalle copie di backup in un'altra cartella. Tuttavia, in caso di malfunzionamenti del server o crash del volume, i dati, così come le copie di backup, potrebbero andare perse. Pertanto, si consiglia di eseguire sempre il backup dei dati importanti su un dispositivo di archiviazione esterno, invece che sullo stesso server. eCore Server consente agli utenti di impostare una destinazione backup non solo sui dischi locali, ma anche su dispositivi USB esterni. La funzione è disponibile nel Menu principale > Backup e replica. Per le istruzioni dettagliate, consultare qui.

2. Backup su un altro eCore Server presente in rete

Il backup dei dati su un dispositivo di archiviazione esterno consente di migliorare la disponibilità dei dati, tenendo anche in considerazione il rischio di disastri come incendi o terremoti che si possono verificare nel luogo dove si trova il server o il dispositivo di archiviazione. eCore Server consente agli utenti di impostare un backup esterno per il trasferimento delle copie dei dati su un server in un altro luogo tramite la rete, riducendo in questo modo il rischio di perdita dei dati causato da disastri nel luogo in cui si trova il server. È possibile crittografare il trasferimento dati del backup in modo da proteggere i dati importanti con un livello di protezione molto alto. Grazie all'opzione di backup a livello blocco, il NAS esegue il backup dei soli blocchi modificati, è utile in particolare quando il backup viene eseguito tramite Internet. La funzione è disponibile nel Menu principale > Backup e replica. Per le istruzioni dettagliate, consultare qui.

3. Backup su spazi di archiviazione cloud pubblici

Il backup remoto tra diversi dispositivi NAS richiede due o più server, un certo numero di dischi in essi contenuti e i luoghi per l'individuazione dei server. La configurazione aziendale potrebbe non consentire l'impostazione di backup esterni con diversi dispositivi NAS. Per offrire una soluzione al problema di una distribuzione impraticabile dei NAS per il backup di rete, la funzione Backup di Synology offre il supporto per diversi servizi cloud pubblici che consentono di eseguire il backup dei dati direttamente su un account cloud pubblico. La funzione è disponibile in Backup e replica o Centro pacchetti in relazione allo spazio di archiviazione cloud preferito. Per le istruzioni dettagliate, consultare qui.
