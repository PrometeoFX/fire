Split-brain

Quando un cluster high-availability funziona normalmente, solamente uno dei server membri può assumere il ruolo di server attivo. In questo caso, il server passivo rileva la presenza del server attivo mediante connessione Heartbeat e connessione dati.

Se tutte le connessioni Heartbeat e dati sono perse, entrambi i server potrebbero tentare di assumere il ruolo di server attivo. Questa situazione prende il nome di errore "split-brain". In questo caso, le connessioni verso l'indirizzo IP assegnato al cluster high-availability saranno reindirizzate a uno dei due server, generando errori nel servizio.

Nel caso di errori split-brain:

  1. Le connessioni verso l'indirizzo IP del cluster saranno indirizzate a entrambi i server, generando errori nel servizio. Si verificheranno difficoltà nella connessione su entrambi i server.
  2. Il sistema rileva l'errore split-brain e rimuove il cluster high-availability, eseguendo l'unbinding di entrambi i server.
  3. Una volta che entrambi i server hanno lo stato Standalone, sarà possibile accedere a DiskStation Manager (DSM). Allo stesso tempo, tutti i dati accumulati prima della rimozione del cluster high-availability rimarranno su entrambi i server. Una volta identificato quale server contiene i dati più aggiornati, è possibile eseguire nuovamente i binding dei server per creare un cluster high-availability.
