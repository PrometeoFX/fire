Sauvegarde des données

Dans l'onglet Sauvegarde de Sauvegarde et réplication, vous pouvez sauvegarder des données spécifiques sur le DiskStation manuellement ou selon un planning personnalisé. Avec cette fonction, la configuration du système, le contenu des dossiers partagés et les données du paquet peuvent être sauvegardées sur une destination de sauvegarde locale, un réseau ou le cloud.

Pour créer une tâche de sauvegarde :

  1. Choisissez Créer > Tâche de sauvegarde des données.
  2. Suivez l’assistant pour terminer la configuration. Visitez Destination de la sauvegarde pour plus d'informations.

Pour gérer les tâches de sauvegarde :

Sélectionnez une tâche existante et cliquez sur l'un des boutons ci-dessous :

  • Modifier : Modifie la destination, la source, le planning et d'autres paramètres de la tâche sélectionnée.
  • Supprimer : Supprime la tâche sélectionnée.
  • Annuler : Annule la tâche sélectionnée en cours.
  • Sauvegarder maintenant : Accomplit immédiatement la sauvegarde de la tâche sélectionnée.

Rotation des sauvegardes

La rotation des sauvegardes est un mécanisme de gestion des versions. Elle contribue à gérer les versions de sauvegarde en supprimant automatiquement les anciennes versions inutiles et en libérant potentiellement de l'espace de stockage. Vous pouvez activer cette fonction dans Paramètres de tâche ou la configurer lorsque vous créez une tâche de sauvegarde.

Pour la rotation des versions :

  1. Sélectionnez la tâche de sauvegarde à laquelle vous souhaitez appliquer la rotation des versions et cliquez sur Modifier.
  2. Accédez à l'onglet Paramètres de tâche et cochez Activer la rotation des sauvegardes.
  3. Vous pouvez ici choisir l'une des deux méthodes de rotation :
    • À partir des versions les plus anciennes : Il s'agit d'une méthode de base de rotation des versions. Lorsque la condition de rotation est déclenchée, la version la plus ancienne est supprimée en premier.
    • Smart Recycle : Il s'agit d'une méthode intelligente de rotation des versions. Lorsque la condition de rotation est déclenchée, le système supprime les versions sélectionnées par le mécanisme de conservation. Le mécanisme de conservation se présente comme suit :
      • Les versions horaires depuis les dernières 24 heures : Le système conserve la version la plus récente créée à chaque heure.
      • Versions quotidiennes de la veille à un mois : Le système conserve la version la plus récente créée chaque jour.
      • Les versions hebdomadaires de plus de 1 mois : Le système conserve la version la plus récente créée chaque semaine.
  4. Indiquez le nombre de versions maximal. La rotation des versions commence lorsque le nombre de versions dépasse ce nombre. Le nombre ne comprend pas les versions verrouillées.
  5. Si vous avez configuré un calendrier de sauvegarde à répétition, un aperçu de l'estimation de la manière dont les versions de sauvegarde seront conservées s'affiche ainsi que le moment le plus ancien de récupération de vos paramètres de rotation.   
  6. Cliquez sur OK pour enregistrer vos paramètres.

Remarques :

  • Les versions verrouillées ne sont jamais supprimées. Visitez Destination de la sauvegarde pour plus d'informations.
  • Seules les tâches de sauvegarde multi-version offrent des fonctions de rotation des versions.
  • La rotation des versions n'est pas prise en charge dans DSM 5.1 et les versions antérieures.
  • La rotation est déclenchée aussitôt après la modification du réglage ou une fois la tâche de sauvegarde terminée.
  • Les versions verrouillées sont exclues lors du calcul du nombre maximum de versions.

Notes générales :

  • Le nom des tâches de sauvegarde peut inclure de 1 à 32 caractères Unicode mais ne peut pas contenir les caractères suivants :
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • La longueur maximale du chemin d'accès complet au dossier pour la sauvegarde est de 2048 caractères.
  • Si plus de 32000 fichiers ou dossiers sont chargés vers un dossier partagé EXT3 via AFP, toutes les tâches de sauvegarde y compris le dossier partagé échoueront.
  • Il est possible de sauvegarder un dossier chiffré monté ou démonté. Toutefois, il est toujours restauré démonté et n'est pas automatiquement monté au démarrage. Vous serez invité à entrer ou à importer la clé de chiffrage du dossier partagé chiffré pour le monter.
  • La sauvegarde d'un dossier chiffré vers un autre dossier chiffré n'est pas prise en charge.
  • Utilisez les dossiers à distance comme source de sauvegarde, sinon la destination de sauvegarde ne sera pas prise en charge.
  • Si la destination de la sauvegarde est un volume, seul le propriétaire de la tâche pourra effectuer la tâche de sauvegarde. Visitez Destination de la sauvegarde pour plus d'informations.
  • La sauvegarde pour certaines applications est prise en charge. Pour voir si un paquet spécifique prend en charge la sauvegarde, ainsi que le contenu et les limitations de la sauvegarde, veuillez consulter l'aide de chaque application.
  • DiskStation ne prend pas en charge la sauvegarde des applications sans appareil externe EXT3/EXT4 inscriptible.
  • DiskStation ne prend pas en charge la sauvegarde vers un volume sans périphérique externe EXT3/EXT4 inscriptible.
  • DiskStation peut uniquement sauvegarder des dossiers partagés créés sur des appareils externes EXT3/EXT4.
  • Le système ne peut pas sauvegarder des applications sans un volume inscriptible.

Sauvegarde des paramètres Windows ACL :

Les paramètres Windows ACL des fichiers enregistrés sur le DiskStation peuvent également être sauvegardés selon les systèmes de fichiers de la destination :

  • La Destination de Sauvegarde Locale, la Destination de Sauvegarde Réseau et la Destination de Sauvegarde du cloud public peuvent sauvegarder les paramètres de Windows ACL quel que soit le système de fichiers de la destination
  • Synchro du dossier partagé peut synchroniser les paramètres de Windows ACL lorsque le système de fichiers de la destination est EXT4/BTRFS.
