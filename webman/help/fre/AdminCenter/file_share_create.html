Création d'un dossier partagé

Les dossiers partagés sont les répertoires de base où vous pouvez stocker et gérer les fichiers et dossiers sur votre DiskStation. Avant de stocker des fichiers, vous devez créer au moins un dossier partagé dans DSM.

Pour créer un dossier partagé :

  1. Allez dans Panneau de configuration > Dossier partagé et cliquez sur Créer.
  2. Spécifiez un nom et une description pour le dossier partagé.
  3. Sélectionnez un stockage sur lequel créer le dossier partagé (passez cette étape s'il n'y a aucun volume existant).
  4. Modifiez les options suivantes si nécessaire :
    • Cacher ce dossier partagé dans « Mes emplacements réseau » : Empêche le dossier partagé d'apparaître sous « Réseau » dans Windows File Explorer. Cette option n'affecte pas les privilèges d'accès du dossier partagé.

      Remarque :

      Cacher un dossier dans “Favoris réseau” n'affecte pas les droits d'accès. Les droits d'accès du dossier partagé caché demeurent inchangés. Les utilisateurs disposant des droits suffisants peuvent toujours accéder à ce dossier partagé en saisissant « \\nom serveur\nom dossier partagé ».

    • Masquer les sous-dossiers et les fichiers des utilisateurs sans autorisations Lorsque Masquer les sous-dossiers et les fichiers des utilisateurs sans autorisations est activé, si un utilisateur sans privilèges de lecture tente d'accéder à un dossier partagé via le protocole de partage de fichiers de Windows, l'utilisateur ne pourra pas voir les sous-dossiers ou les fichiers dans le dossier partagé. Cela empêche les utilisateurs d'être embrouillés quand ils se connectent à leurs produits Synology et trouvent un grand nombre de fichiers et de sous-dossiers qu'ils ne peuvent pas ouvrir.

      Remarque :

      • Cette fonctionnalité est uniquement prise en charge avec le protocole de partage de fichiers de Windows.
      • Vous pouvez modifier les paramètres de privilège de chaque dossier dans File Station en cliquant dessus avec le bouton droit de la souris. Choisissez Propriétés dans le menu déroulant et cliquez sur l'onglet Permission.
      • Le dossier partagé lui-même ne sera pas caché aux utilisateurs sans autorisation en activant Masquer les sous-dossiers et les fichiers aux utilisateurs sans autorisations. Seuls les sous-dossiers et les fichiers qui se trouvent dans ce dossier partagé sont masqués.
  5. Activer la corbeille :Quand les fichiers dans le dossier partagé sont supprimés, ils seront déplacés vers un dossier nommé #recycle L'accès à la corbeille de recyclage peut être limité aux utilisateurs appartenant au groupe administrators.
  6. Rendre l'instantané visible : Il est possible d'accéder aux dossier et aux fichiers qui se trouvent dans des instantanés via des services de fichiers dans des dossiers #snapshot du dossier partagé. Les instantanés sont nommés en fonction de l'heure de Greenwich (GMT) au moment où ils sont pris.

    Remarque :

    • Les utilisateurs ne disposent de permission en écriture sur aucune copie d'instantané.
    • Les utilisateurs ont une permission en lecture uniquement sur la copie de l'instantané si, au moment où l'instantané est pris, leurs comptes utilisateurs ont été créés et qu'on leur a octroyé la permission en lecture sur le dossier partagé.
  7. Activer le quota du dossier partagé : Pour indiquer la capacité maximale de chaque dossier partagé, cochez la case Activer le quota du dossier partagé et saisissez la capacité maximale (par ex., 10 Go) dans le champ suivant.

    Remarque :

    • Les options de Rendre l'instantané visible et Activer le quota du dossier partagé ne sont disponibles pour les dossiers partagés que lorsque le stockage utilise le système de fichiers Btrfs.
    • L'utilisation du stockage combiné des répertoires home de tous les utilisateurs ne peut pas dépasser le quota de « homes ».
  8. Activer la protection avancée de l'intégrité des données : Pour protéger un dossier partagé avec les stratégies CRC32 checksum et copie lors de l'écriture, vous pouvez activer la protection avancée de l'intégrité des données lors de la création du dossier partagé. CRC32 checksum est appliqué afin de vérifier si les données demeurent telles qu'elles ont été écrites à l'origine et notifie l'utilisateur lors de la détection d'une altération. La copie lors de l'écriture permet d'améliorer la cohérence des données en cas d'arrêt anormal du système.

    Remarque :

    • Cette option est uniquement disponible pour les volumes utilisant le système de fichiers Btrfs.
    • Vous pouvez activer ou désactiver cette option pour chaque dossier partagé lors de sa création. Il est impossible de modifier cette option une fois le dossier partagé créé.
    • Les performances du système peuvent être affectées si l'option est activée.
  9. Chiffrer ce dossier partagé : Si vous souhaitez chiffrer le dossier partagé, procédez comme suit :
    1. Spécifiez et confirmez une nouvelle clé de chiffrement. La clé de chiffrement ne peut pas contenir des virgules (,) ou le signe égal (=).
    2. Si nécessaire, cochez la case Monter automatiquement au démarrage pour monter le dossier chiffré automatiquement quand le système démarre. Si cette option n'est pas désactivée, le dossier crypté partagé est déconnecté au prochain démarrage de eCore OS. Pour afficher ou accéder à un dossier partagé déconnecté, vous devrez monter le dossier manuellement en important ou en entrant la clé de chiffrement.
  10. Cliquez sur OK pour terminer.

Règles de nommage du dossier partagé :

  • Les noms de dossiers partagés sont sensibles à la casse et peuvent se composer de 1 à 32 caractères. Les noms ne peuvent pas inclure les caractères suivants : ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Les noms suivants sont réservés pour le système : ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Le premier caractère du nom d'un dossier partagé ne peut être ni le signe moins ni un espace, tout comme le dernier caractère qui ne peut pas non plus être un espace.
  • La description du dossier partagé est sensible à la casse et est limitée à 64 caractères Unicode affichables.

Dossiers partagés chiffrés :

Le chiffrage AES 256 bits protège les données en bloquant toute tentative d'accès non autorisée. Quand un dossier partagé est chiffré, ceux qui ne possèdent pas la clé de chiffrement ne peuvent pas accéder ou modifier les données cryptées même s'ils retirent les disques durs de votre DiskStation et les montent sur un autre appareil.

Remarque :

  • Les dossiers partagés peuvent uniquement être chiffrés au moment de la création.
  • Les noms des fichiers et dossiers dans le dossier partagé ne peuvent pas dépasser 143 caractères latins ou 43 caractères asiatiques.
  • Il n'est pas possible d'accéder aux dossiers partagés cryptés via NFS.
  • Si vous restaurez les mots de passe par défaut à l'aide du bouton Réinitialiser sur le DiskStation, les dossiers partagés cryptés sont démontés, et l'option de montage automatique des dossiers partagés cryptés est désactivée.
