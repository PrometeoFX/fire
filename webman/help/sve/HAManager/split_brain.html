Redundans-klyvning

När ett high-availability-kluster fungerar normalt, skall endast en av medlemsservrarna inneha rollen som aktiv server. I detta fall upptäcker den passiva servern förekomsten av den aktiva servern både via Heartbeat-anslutningen och dataanslutningen.

Om alla Heartbeat- och dataanslutning går förlorad kan det hända att båda servrarna försöker anta roller som aktiv server. Denna situation kallas för "redundansklyvning". När så är fallet kommer anslutningar till IP-adressen som tilldelats high-availability-klustret att omdirigeras till vilken som av de två servrarna, vilket resulterar i felaktiga tjänster.

Om redundansklyvning uppstår:

  1. Anslutningar till IP-adressen som tilldelats high-availability-klustret kommer att omdirigeras till vilken som av de två servrarna, vilket resulterar i felaktiga tjänster. Du upplever problem med att ansluta till vilken som av servrarna.
  2. Systemet kommer att upptäcka redundansklyvningen och ta bort high-availability-klustret samt häva bindningen av de båda servrarna.
  3. När båda servrarna återgår till status som Fristående, kommer du att kunna logga in på DiskStation Manager (DSM). I detta läge kommer data som ackumulerats före high-availability-klustret togs bort att finnas kvar på båda servrarna. Efter att du identifierat den server som har den mest uppdaterade informationen kan du åter binda servrarna för att skapa ett high-availability-kluster igen.
