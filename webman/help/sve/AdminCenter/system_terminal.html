Terminal

Aktivering av alternativ i Kontrollpanel > Terminal & SNMP > Terminal ger din DiskStation-enhet stöd för tjänsterna Telnet och SSH med kommandoradsgränssnitt. Du kan även ändra säkerhetsnivån för SSH-krypteringsalgoritmen.

Aktivera Telnet-/SSH-tjänsten:

  1. Markera rutan bredvid det önskade protokollet (SSH eller Telnet).
  2. Klicka på Tillämpa.

För att ändra port för SSH-tjänsten:

  1. I Port, ange ett nytt portnummer.
  2. Klicka på Tillämpa.

Ett urval av krypteringsalgoritmer med olika säkerhetsnivåer används i säkra SSH-anslutningar. Du kan avgöra vilka algoritmer som bäst motsvarar dina behov.

För att ändra säkerhetsnivån på SSH-krypteringsalgoritm:

  1. Klicka på Avancerade inställningar.
  2. Välj en säkerhetsnivå för krypteringsalgoritmer. Om du vill specificera exakt vilka algoritmer som kan användas, välj Anpassad för att välja från tre grupper: Cipher, KEX (Key Exchange) och MAC (Message Authentication Code).
  3. Klicka på Tillämpa.

En del modeller har stöd för hårdvaruacceleration för vissa krypteringsalgoritmer. Du kan markera alternativet Använd bara hårdvaruaccelererat skiffer för att förbättra överföringshastigheten för SSH-relaterade tjänster, inklusive krypterad nätverkssäkerhetskopiering och SFTP-tjänster. På den avancerade sidan visas algoritmer som kan använda hårdvaruacceleration i rött.

Obs:

  • En del tjänster (t.ex nätverkssäkerhetskopiering, SFTP) aktiverar automatiskt SSH och öppnar port 22. I detta fall måste du aktivera tjänsten SSH i Kontrollpanel > Terminal & SNMP > Terminal för att logga in på DSM med SSH.
  • Det finns endast stöd för SSH/Telnet vid inloggning i systemet med kontona root eller admin.
  • Lösenordet kan inte vara tomt vid inloggning i systemet med SSH/Telnet.
  • När tjänsten SSH är avaktiverad, avslutas alla SSH-anslutningar, inklusive alla krypterad säkerhetskopieringsuppgifter som för närvarande körs. Vi föreslår att nätverkssäkerhetskopieringsuppgift utförs igen efter att SSH avaktiverats.
  • Felaktig användning eller ändring av DiskStation kan leda till funktionsstörning eller dataförlust.
  • Val av en högre säkerhetsnivå eller att bara använda hårdvaruaccelererade skiffer kan orsaka att en del SSH-klienter inte kan ansluta beroende på kompatibilitetsproblem.
  • Beroende på den valda säkerhetsnivån är vissa algoritmer som stöder hårdvaruacceleration eventuellt inte tillgängliga.
