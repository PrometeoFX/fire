Visión general

Esta página muestra información básica sobre el clúster high-availability y sus servidores. Además, esta página ofrece herramientas para crear, eliminar o realizar tareas de mantenimiento en clústers high-availability y en sus servidores.

Estado de clúster high-availability

Este campo muestra el estado general de su clúster high-availability y le avisa de cualquier error. A continuación se muestra una tabla de los estados que pueden aparecer en este campo:

Estado Descripción
Independiente Los servidores no están combinados en un clúster high-availability.
En buen estado El estado general de su clúster high-availability es bueno. La replicación de datos del servidor activo al servidor pasivo funciona normalmente.
Preparando Los servidores se han combinado correctamente en un clúster high-availability, pero aún no ha finalizado la replicación de datos inicial. El cambio no se puede realizar en este momento. Para más información, consulte Cambio.
Cargando El sistema está cargando el estado del clúster. No se puede realizar un Cambio hasta que no se haya completado la carga del estado.
Advertencia El estado del clúster HA es anormal, pero los servicios siguen funcionando.
Peligro Tanto el servidor activo como el pasivo funcionan mal. El clúster high-availability no puede funcionar normalmente. Compruebe el estado de todos los espacios de almacenamiento (p. ej., RAID Groups, Grupos de discos, volúmenes e iSCSI LUNs).

Mensajes de advertencia

La siguiente tabla muestra los mensajes de advertencia que pueden aparecer cuando se produce un error en el sistema. Cuando aparezca más de un mensaje de advertencia, puede desplazarse entre ellos con los botones de avance/retroceso.

Advertencia/Error Descripción
Error de servicio controlado Se ha producido un error en uno o más servicios controlados.
Espacio de almacenamiento degradado Un espacio de almacenamiento (p. ej., RAID Group, Grupo de discos, volumen, iSCSI LUN y caché de SSD) del servidor activo o pasivo está degradado. Consulte la página Estado de almacenamiento.
Espacio de almacenamiento averiado Un espacio de almacenamiento (p. ej., RAID Group, Grupo de discos, volumen, iSCSI LUN y caché de SSD) del servidor activo o pasivo está averiado. Consulte la página Estado de almacenamiento.
No se ha podido detectar el servidor pasivo Este error podría producirse porque el servidor pasivo está desconectado o los servidores activo y pasivo no se pueden conectar a través de la conexión Heartbeat. En este caso, el cambio no estará disponible.
Los datos no se pueden replicar en el servidor pasivo Este error podría deberse a un fallo en el volumen de almacenamiento o en el disco. En este caso, el cambio no estará disponible. Consulte la página Estado de almacenamiento.
El servidor pasivo no se ha seleccionado Si el servidor pasivo no está vinculado al clúster high-availability, se mostrará este estado. En este caso, el cambio no estará disponible. Si desea volver a vincular el servidor pasivo, haga clic en el menú desplegable Mantener y luego haga clic en Vincular. Cuando vuelva a vincular el servidor pasivo, será necesaria una replicación completa de los datos.

Información del clúster high-availability

Este campo se encuentra debajo del estado del clúster high-availability y muestra información básica sobre el clúster. Si no existe un clúster high-availability, este campo solo mostrará una explicación de la función de los clústers high-availability.

En esta campo se muestra la siguiente información:

  • Nombre de servidor de clúster: el nombre de servidor que usa el clúster high-availability.
  • Dirección IP de clúster: la dirección IP que usa el clúster high-availability. Independientemente del servidor que esté ejecutando los servicios, los usuarios pueden conectarse al clúster high-availability utilizando esta dirección IP.
  • Tiempo acumulado: la fecha cuando se creó el clúster high-availability.

Botones de funcionamiento del clúster high-availability

Los dos botones siguientes se encuentran debajo de la información del clúster high-availability:

  1. Crear clúster HA: cuando no existe ningún clúster high-availability, puede hacer clic en este botón para combinar dos servidores y crear un clúster high-availability. Para más información sobre cómo crear un clúster high-availability, consulte el Asistente de Synology High Availability.
  2. Gestión: este botón solo está disponible cuando ya existe un clúster high-availability. Incluye las siguientes opciones:
    • Asistente de gestión: Abre un asistente que incluye instrucciones para distintos procedimientos de gestión.
    • Cambio: inicia un cambio de servicios del servidor activo al servidor pasivo. Durante este proceso, el servidor activo deja de suministrar servicios y se convierte en el nuevo servidor pasivo. Entonces, el servidor pasivo anterior pasa a ser el nuevo servidor activo y empieza a suministrar todos los servicios del clúster high-availability. Esta opción no está disponible cuando el servidor pasivo está en estado de error o no está seleccionado.
    • Actualizar DSM: abre la página Actualizar DSM del Panel de control. Desde aquí se pueden actualizar ambos servidores.
    • Eliminar clúster HA: elimina el clúster high-availability.
    • Desvincular el servidor pasivo: desvincula el servidor pasivo del clúster. Cuando se vuelva a añadir el servidor al clúster high-availability después de desvincularlo, será necesario llevar a cabo una replicación de todos los datos. Por tanto, le recomendamos que únicamente utilice esta opción cuando quiera sustituir el servidor pasivo existente por uno nuevo.
    • Vincular servidor pasivo: agrega un servidor pasivo al clúster. Esta opción solo está disponible cuando no hay un servidor pasivo en el clúster de high-availability.
    • Apagar el clúster HA: apaga los servidores activos y pasivos.
    • Apagar el servidor activo: apaga el servidor activo.
    • Apagar el servidor pasivo: apaga el servidor pasivo.
    • Reiniciar el servidor pasivo: reinicia el servidor pasivo.
    • Desactivar el pitido en el servidor pasivo: detiene el sonido del pitido del servidor pasivo cuando se produce un error. El servidor pasivo volverá a pitar si se produce otro error más tarde.

Nota:

  • Una vez que se elimina un clúster high-availability, será necesario realizar una replicación de los todos datos cuando desee crear uno nuevo.
  • Si apaga o desvincula el servidor pasivo, los servicios no se pueden cambiar si se produce un error en el servidor activo.

Estado de los servidores y de la replicación de los datos

Esta sección contiene imágenes que indican el estado de los servidores activo y pasivo, así como el flujo de la replicación de los datos entre los servidores.

La imagen de la replicación de los datos indica los posibles estados:

  • La replicación de los datos funciona normalmente: lo indica el paso de los datos del servidor activo al servidor pasivo.
  • Sin replicación de datos: lo indica la ausencia de la imagen del flujo de datos.

Información del servidor

Esta sección se encuentra debajo de las imágenes de los servidores. En esta sección se muestra información sobre los servidores activo y pasivo. Si no se ha seleccionado ningún servidor pasivo, en esta sección aparecerá N/A.

  • Nombre del servidor: nombre del servidor de Synology.
  • Nombre de modelo: modelo del servidor de Synology.
  • Número de serie: número de serie del servidor de Synology.
  • Estado del ventilador: estado de los ventiladores de refrigeración de un servidor. Si un ventilador falla, aparecerá una advertencia. En este caso, póngase en contacto con su distribuidor local para obtener ayuda.
  • Temperatura: temperatura del servidor de Synology.
  • Estado de energía: indica si todas las fuentes de alimentación de un servidor funcionan normalmente o no. Si el modelo del servidor cuenta con fuentes de alimentación redundantes, aparecerá una advertencia si una de ellas no funciona correctamente.
