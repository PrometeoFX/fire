Copia de seguridad de datos

En la pestaña Copia de seguridad de Copia de seguridad y replicación, puede realizar copias de seguridad de datos en DiskStation manualmente o de acuerdo con un programa personalizado. Con esta característica se pueden realizar copias de seguridad de la configuración del sistema, del contenido de las carpetas compartidas y de los datos del paquete a un destino de copia de seguridad local, de red o en la nube.

Para crear una tarea de copia de seguridad:

  1. Elija Crear > Tarea de copia de seguridad de datos.
  2. Siga el asistente para completar la configuración. Visite Destino de copia de seguridad para obtener más información.

Para administrar tareas de copia de seguridad:

Seleccione una tarea existente y haga clic en uno de los botones siguientes:

  • Editar: Para modificar el destino, el origen, el programa y otras configuraciones de la tarea seleccionada.
  • Eliminar: Para eliminar la tarea seleccionada.
  • Cancelar: Para cancelar la tarea seleccionada que se está ejecutando actualmente.
  • Realizar copia de seguridad ahora: Realiza una copia de seguridad de la tarea seleccionada inmediatamente.

Rotación de copia de seguridad

La rotación de copia de seguridad es un mecanismo de administración de versiones. Ayuda a administrar las versiones de copias de seguridad mediante la eliminación automática de versiones anteriores no necesarias y así potencialmente liberar espacio de almacenamiento. Puede habilitar esta función en Configuración de tarea o configurarla cuando cree una tarea de copia de seguridad.

Para rotar versiones:

  1. Seleccione la tarea a la que desea aplicar la rotación de versiones y haga clic en Editar.
  2. Vaya a la pestaña Configuración de tarea y marque Habilitar la rotación de copias de seguridad.
  3. Aquí puede elegir uno de los dos métodos de rotación:
    • A partir de la versión más antigua: Este es un método de rotación básico. Cuando se active la condición de rotación, la versión más antigua se eliminará primero.
    • Smart Recycle: Este es un método de rotación inteligente. Cuando se active la condición de rotación, el sistema eliminará las versiones seleccionadas por el mecanismo de retención. El mecanismo de retención funciona de la siguiente manera:
      • Versiones cada hora desde las últimas 24 horas: El sistema conserva la versión más antigua creada cada hora.
      • Versiones diarias desde hace 1 día a hace 1 mes: El sistema conserva la versión más antigua creada cada día.
      • Versión semanal después de 1 mes: El sistema conserva la versión más antigua creada cada semana.
  4. Indique el número máximo de versiones. Cuando el número de versiones supera este número, la rotación de versiones se iniciará. El número no incluye las versiones bloqueadas.
  5. Si ha configurado un programa de copia de seguridad repetitivo, se conservará una vista previa del cálculo de la manera de retener versiones de copias de seguridad y se mostrará la hora del punto de recuperación más antiguo basado en la configuración de la rotación.   
  6. Haga clic en OK para guardar su configuración.

Notas:

  • Las versiones bloqueadas no se eliminarán nunca. Visite Destino de copia de seguridad para obtener más información.
  • Solo las tareas de copia de seguridad de varias versiones proporcionan funciones de rotación de versiones.
  • La rotación de versiones no es compatible con DSM 5.1 o versiones inferiores.
  • La rotación se activará inmediatamente después de que la configuración se cambie o de que se complete la tarea de copia de seguridad.
  • Las versiones bloqueadas quedan excluidas cuando se calcula el número máximo de versiones.

Notas generales:

  • Los nombres de tareas pueden incluir entre 1 y 32 caracteres Unicode, pero no pueden contener los siguientes caracteres:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • La longitud máxima de la ruta de carpeta completa para copias de seguridad es de 2048 caracteres.
  • Si se cargan más de 32 000 archivos o carpetas en una carpeta compartida EXT3 mediante AFP, todas las tareas de copia de seguridad, incluida la carpeta compartida, producirán errores.
  • Se puede hacer copia de seguridad de una carpeta cifrada montada o desmontada. Sin embargo, siempre se restaurará como desmontada y no se montará automáticamente al arrancar. Deberá introducir o importar la clave de cifrado de la carpeta compartida cifrada para que se monte.
  • Realizar copias de seguridad de una carpeta cifrada a otra carpeta cifrada no es compatible.
  • Utilizar carpetas remotas como origen o como destino de copia de seguridad no es compatible.
  • Si el destino de copia de seguridad es un volumen, solo el propietario de la tarea podrá realizar la tarea de la copia de seguridad. Visite Destino de copia de seguridad para obtener más información.
  • La realización de copias de seguridad en algunas aplicaciones es compatible. Para comprobar si un paquete específico es compatible con la realización de copias de seguridad, además del contenido de dichas copias y sus limitaciones, consulte la sección de ayuda de cada aplicación.
  • DiskStation no es compatible con la realización de copias de seguridad para aplicaciones sin un dispositivo externo EXT3/EXT4 grabable.
  • DiskStation no es compatible con la realización de copias de seguridad a volúmenes sin un dispositivo externo EXT3/EXT4 grabable.
  • DiskStation sólo puede realizar copias de seguridad de carpetas compartidas creadas en dispositivos externos EXT3/EXT4.
  • El sistema no puede realizar copias de seguridad de aplicación sin un volumen grabable.

Copias de seguridad de la configuración ACL de Windows:

También se pueden realizar copias de seguridad de la configuración ACL de Windows de los archivos en DiskStation dependiendo de los sistemas de archivos en el destino:

  • Destino de copia de seguridad local, Destino de copia de seguridad de red y Destino de copia de seguridad en nube pública pueden realizar una copia de seguridad de la configuración Windows ACL, independientemente del sistema de archivos del destino
  • Sincronización de carpetas compartidas puede sincronizar la configuración ACL de Windows cuando el sistema de archivos del destino sea EXT4/BTRFS.
