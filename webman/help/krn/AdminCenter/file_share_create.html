공유 폴더 생성

공유 폴더는 DiskStation에 파일과 폴더를 저장하고 관리할 수 있는 기본 디렉토리입니다. 파일을 저장하기 전에 DSM에 최소 하나의 공유 폴더를 생성해야 합니다.

공유 폴더 생성하기:

  1. 제어판 > 공유 폴더로 이동하여 생성을 클릭합니다.
  2. 공유 폴더의 이름과 설명을 지정합니다.
  3. 공유 폴더를 만들 저장소를 선택합니다(저장소가 하나만 있을 경우 이 단계는 건너뜀).
  4. 필요한 경우 다음 옵션을 수정합니다:
    • 이 공유 폴더를 "내 네트워크 환경"에서 숨기기: Windows 파일 탐색기에서 "네트워크" 아래에 해당 공유 폴더가 표시되지 않도록 합니다. 이 옵션은 공유 폴더의 접근 권한에 영향을 미치지 않습니다.

      참고:

      "내 네트워크 환경"에서 공유 폴더를 숨겨도 접근 권한에는 영향을 주지 않습니다. 숨은 공유 폴더의 접근 권한은 변하지 않습니다. 공유 폴더에 대한 적절한 접근 권한을 가진 사용자는 "\\서버 이름\공유 폴더 이름"을 입력해서 계속 공유 폴더에 접근할 수 있습니다.

    • 권한 없는 사용자에게 하위 폴더 및 파일 숨기기: 권한 없는 사용자에게 하위 폴더 및 파일 숨기기를 활성화할 경우 읽기 권한이 없는 사용자가 Windows 파일 공유 프로토콜을 통해 공유 폴더에 액세스하면 공유 폴더 안에 있는 폴더나 파일이 보이지 않게 됩니다. 이 기능은 사용자가 Synology 제품에 연결할 때 자신이 열 수 없는 많은 파일과 하위 폴더를 확인하는 번거로움을 덜어줍니다.

      참고:

      • 이 기능은 Windows 파일 공유 프로토콜을 통해서만 지원됩니다.
      • File Station에서 마우스 오른쪽 단추를 클릭하여 각 폴더와 파일의 권한 설정을 수정할 수 있습니다. 드롭다운 메뉴에서 속성을 선택하고 권한 탭을 클릭합니다.
      • 권한 없는 사용자에게 하위 폴더 및 파일 숨기기를 활성화하여 권한이 없는 사용자에게 공유 폴더 자체를 숨기지는 못합니다. 이 공유 폴더 안에 있는 하위 폴더와 파일만 숨겨집니다.
  5. 휴지통 활성화: 공유 폴더에서 파일을 삭제하면 파일이 #recycle 폴더로 이동됩니다. administrators 그룹에 속하는 사용자로 휴지통에 대한 액세스를 제한할 수 있습니다.
  6. 스냅샷을 보이게 만들기: 공유 폴더의 #snapshot 폴더에서 파일 서비스를 통해 스냅샷의 폴더와 파일에 액세스할 수 있습니다. 스냅샷에는 그리니치 평균시(GMT)로 나타낸 촬영 시간이 이름으로 사용됩니다.

    참고:

    • 사용자가 스냅샷 사본에 대한 쓰기 권한은 갖지 않습니다.
    • 스냅샷 촬영 당시 사용자 계정이 생성되었고 사용자에게 공유 폴더에 대한 읽기 권한이 부여된 경우에만 사용자가 스냅샷 복사본에 대한 읽기 권한을 가집니다.
  7. 공유 폴더 할당량 활성화: 각 공유 폴더의 최대 용량을 지정하려면, 공유 폴더 할당량 활성화 선택 상자를 선택하고, 그 다음 필드에 최대 용량을 입력합니다(예: 10 GB).

    참고:

    • 저장소가 Btrfs 파일 시스템을 사용하는 경우에만 스냅샷을 보이게 만들기 및 공유 폴더 할당량 활성화 옵션을 공유 폴더에 대해 사용할 수 있습니다.
    • 모든 사용자의 home 디렉터리를 결합한 저장소 사용량은 “homes”의 할당량을 초과할 수 없습니다.
  8. 고급 데이터 무결성 보호 활성화: CRC32 체크섬 및 쓰기 시 복사 전략으로 공유 폴더를 보호하기 위해, 공유 폴더를 생성하는 동안 고급 데이터 무결성 보호 활성화할 수 있습니다. CRC32 체크섬은 데이터가 원래 쓰여질 때와 동일하게 남아 있는지 확인하는 데 적용되며, 손상이 감지되면 사용자에게 알립니다. 쓰기 시 복사는 비정상적인 시스템 종료 시 데이터 일관성을 향상하는 데 도움이 됩니다.

    참고:

    • 이 옵션은 Btrfs 파일 시스템을 사용하는 볼륨에만 사용할 수 있습니다.
    • 공유 폴더를 생성하는 동안 각 공유 폴더에 대해 이 옵션을 활성화/비활성화할 수 있습니다. 공유 폴더가 일단 생성되고 나면 이 옵션을 변경할 수 없습니다.
    • 이 옵션을 활성화하면 시스템 성능에 영향을 줄 수 있습니다.
  9. 이 공유 폴더 암호화: 공유 폴더를 암호화하려면 다음을 수행하십시오.
    1. 새 암호화 키를 지정하고 확인합니다. 암호화 키에는 쉼표(,) 또는 등호(=)를 포함할 수 없습니다.
    2. 필요하면, 시작 시 자동 마운트 상자를 선택하여 시스템이 시작할 때 암호화된 폴더를 자동으로 마운트합니다. 이 옵션을 비활성화하면 다음 번 DiskStation이(가) 다시 시작할 때 암호화된 공유 폴더의 마운트가 해제됩니다. 마운트가 해제된 공유 폴더를 보거나 접근하려면 암호화 키를 가져오거나 입력해서 폴더를 수동으로 마운트해야 합니다.
  10. 확인을 클릭하여 종료합니다.

공유 폴더 명명 규칙:

  • 공유 폴더 이름은 대소문자를 구분하지 않으며 1~32자여야 합니다. 이름에 다음 문자는 포함할 수 없습니다: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • 다음 이름은 시스템 용도로 예약되어 있습니다: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • 공유 폴더 이름의 첫 번째 문자는 빼기(-) 기호 또는 빈칸일 수 없으며 마지막 문자는 빈칸으로 둘 수 없습니다.
  • 공유 폴더 설명은 대소문자를 구분하며 최대 64자의 표시 가능한 Unicode 문자로 한정됩니다.

공유 폴더 암호화:

AES 256 비트 암호화는 승인되지 않은 모든 접근 시도를 차단하여 데이터를 보호합니다. 공유 폴더가 암호화되어 있을 경우, 암호화 키가 없는 사용자는 DiskStation에서 해당 하드 드라이브를 제거해서 다른 장치에 마운트한 경우에도 암호화된 데이터에 액세스하거나 수정할 수 없습니다.

참고:

  • 공유 폴더는 이를 생성할 때만 암호화할 수 있습니다.
  • 암호화된 공유 폴더 내 파일과 폴더의 이름은 영문자 143자 또는 아시아 문자 43자를 넘을 수 없습니다.
  • 암호화된 공유 폴더는 NFS 를 통해 접근할 수 없습니다.
  • DiskStation에 있는 RESET 단추를 사용하여 패스워드를 기본값으로 복원하면, 암호화된 공유 폴더가 마운트 해제되고 암호화된 공유 폴더를 자동으로 마운트하는 옵션이 비활성화됩니다.
