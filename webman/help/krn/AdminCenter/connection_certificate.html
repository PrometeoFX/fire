인증서

인증서는 DiskStation의 웹(모든 HTTPS 서비스), 메일 또는 FTP 같은 SSL 서비스를 보호하는 데 사용될 수 있습니다.인증서가 있으면 사용자가 기밀 정보를 보내기 전에 서버의 ID와 관리자를 검증할 수 있습니다.

인증서 탭(제어판 > 보안 > 인증서에 위치)은 DiskStation 인증서의 상태를 관리 및 확인하는 옵션을 제공하여 사용자가 인증서를 생성, 가져오기/내보내기 또는 갱신할 수 있도록 해줍니다.인증서 설정 마법사는 자체 서명 인증서를 생성하거나 인증서 서명 요청(CSR)을 다운로드하여 인증 기관(CA)에 등록할 수 있도록 도와줍니다.

참고:

VPN Server가 설치되어 있으면 OpenVPN Server에서는 여기에 설정된 인증서를 자동으로 사용합니다.

자체 서명 인증서

자체 서명 인증서는 ID를 인증하는 동일한 단체에 의해 생성되고 서명된 인증서를 가리킵니다(이 경우 DiskStation).자체 서명 인증서는 DiskStation에서 생성된 개인 키로 서명합니다.자체 서명 인증서는 제3자 인증 기관이 발급하지 않기 때문에, 서버 ID 증명에 융통성이 없어서 보통 서버와 알려진 사용자 그룹 간에 채널을 보호하는 데만 사용됩니다.

자체 서명 인증서 생성하기:

  1. 인증서 생성을 클릭합니다.
  2. 설정 마법사의 지시를 따릅니다.

참고:

자체 서명 인증서를 생성하면 DiskStation의 기존 인증서가 교체됩니다.

인증서 서명 요청

제3자 인증 기관의 인증서를 신청하려면 먼저 인증서 서명 요청(CSR)을 생성해야 합니다.인증서 서명 요청은 도메인 이름, 조직 이름, 일반 위치 및 이메일 주소 같은 인증서에 넣을 정보를 포함한 암호화된 본문 텍스트로 DiskStation에서 생성됩니다.설정 마법사는 DiskStation에 대해 서명된 인증서를 신청하기 위한 인증서 서명 요청을 생성하도록 도움을 줍니다.

상업 기관 또는 제3자 인증 기관에서 인증서를 얻으려면, 개인이나 조직의 ID를 제공하고 사용자가 인증서 서명 요청의 공통 이름 필드에 입력한 도메인 이름의 소유주가 맞는지 입증해야 합니다.

참고:

인증서 서명 요청과 함께 개인 키도 생성되어야 합니다.인증 기관은 이 개인 키를 필요로 하지 않습니다.DiskStation의 개인 키를 안전하게 잘 보관하십시오.

인증서 서명 요청 생성하기:

  1. 인증서 생성을 클릭합니다.
  2. 설정 마법사의 지침에 따라 인증서 서명 요청을 생성하고 다운로드합니다.

인증 기관에서 인증서를 발행하면 개인 키와 함께 이를 가져올 수 있습니다(지침은 아래 참조).

인증서 내보내기

관리 또는 보관을 위해 기존 인증서를 다운로드할 수 있습니다.내보낸 파일은 DiskStation의 인증서, 개인 키 및 자체 서명 루트 인증서를 포함합니다.

인증서 내보내기:

  • 인증서 내보내기 단추를 클릭합니다.

인증서 가져오기

이 옵션을 사용하여 인증 기관에서 발행한 인증서나 이전에 내보낸 인증서를 개인 키와 함께 가져올 수 있습니다.

인증서 가져오기:

  • 인증서 가져오기 단추를 클릭합니다.

참고:

  • 중간 인증서는 일부 인증 기관이 발행한 인증서의 옵션입니다.
  • 인증서는 X.509 PEM 형식이어야 합니다.
  • 개인 키는 RSA 형식이어야 하며 패스프레이즈로 보호되지 않습니다.

인증서 갱신

인증서가 만료되려고 할 때 이 옵션을 사용하여 갱신할 수 있습니다.

인증서 갱신하기:

  1. 인증서 생성을 클릭합니다.
  2. 인증서 갱신을 선택하고 다음을 클릭합니다.
  3. 새로운 개인 키와 인증서 서명 요청이 생성됩니다.

인증서 서명 요청(CSR)에 서명

이 옵션을 선택하면 DiskStation의 루트 인증서를 사용하여 인증서 서명 요청에 서명할 수 있습니다.

인증서 서명 요청에 서명하기:

  1. 인증서 생성을 클릭합니다.
  2. 인증서 서명 요청(CSR)에 서명을 클릭합니다.
  3. 인증서 서명 요청을 업로드하고 관련 정보를 입력합니다.
  4. 다음을 클릭하면 시스템이 인증서 요청에 서명하고 해당 인증서를 생성합니다.
