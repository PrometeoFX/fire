데이터 백업

백업 및 복제의 백업 탭 아래에서, DiskStation의 특정 데이터를 수동 또는 사용자 지정 일정에 따라 백업할 수 있습니다. 이 기능을 통해, 시스템 구성, 공유 폴더 내용 및 패키지 데이터를 로컬, 네트워크 또는 클라우드 백업 대상으로 백업할 수 있습니다.

백업 작업 생성하기:

  1. 생성 > 데이터 백업 작업을 선택합니다.
  2. 마법사를 따라 설치를 완료하십시오. 자세한 내용은 백업 대상을 참조하십시오.

백업 작업 관리하기:

기존 작업을 선택하고 아래 단추 중 하나를 클릭합니다:

  • 편집: 선택한 작업의 대상, 소스, 일정 및 기타 설정을 편집합니다.
  • 삭제: 선택한 작업을 삭제합니다.
  • 취소: 선택한 현재 실행 중인 작업을 취소합니다.
  • 지금 백업: 선택한 작업의 백업을 즉시 수행합니다.

백업 회전

백업 회전은 버전 관리 메커니즘입니다. 이 기능은 오래되고 불필요한 버전을 자동 삭제하여 잠재적인 저장소 공간을 확보함으로써 백업 버전 관리하는 것을 도와줍니다. 작업 설정에서 이 기능을 활성화하거나 백업 작업을 생성할 때 이를 구성할 수 있습니다.

버전을 회전하려면:

  1. 버전 회전을 적용할 백업 작업을 선택하고 편집을 클릭합니다.
  2. 작업 설정 탭으로 이동하여 백업 회전 활성화를 선택합니다.
  3. 여기에서 두 가지 회전 방법 중 하나를 선택할 수 있습니다:
    • 가장 초기 버전에서: 이것은 기본 버전 회전 방법입니다. 회전 조건이 트리거되면, 가장 초기 버전이 먼저 제거됩니다.
    • Smart Recycle: 이것은 지능형 버전 회전 방법입니다. 회전 조건이 트리거되면, 시스템은 보존 메커니즘에 의해 선택된 버전을 제거합니다. 보존 메커니즘은 다음과 같습니다:
      • 과거 24시간부터 매시간 버전: 시스템이 매 시간 가장 초기에 생성된 버전을 보관합니다.
      • 과거 1일에서 1개월 사이의 매일 버전: 시스템이 매일 가장 초기에 생성된 버전을 보관합니다.
      • 1개월 이상된 매주 버전: 시스템이 매주의 가장 초기에 생성된 버전을 보관합니다.
  4. 최대 버전 수를 지정하십시오. 버전 수가 이 숫자를 초과하면 버전 회전이 시작됩니다. 잠긴 버전은 숫자에 포함되지 않습니다.
  5. 반복적인 백업 일정을 구성한 경우, 회전 설정을 토대로 백업 버전이 보유되는 방식과 가장 빠른 복구 포인트의 미리보기가 표시됩니다.   
  6. 확인을 클릭하여 설정을 저장합니다.

참고:

  • 잠근 버전은 절대로 제거되지 않습니다. 자세한 내용은 백업 대상을 참조하십시오.
  • 복수 버전 백업 작업만이 버전 회전 기능을 제공합니다.
  • DSM 5.1 이하에서는 버전 회전이 지원되지 않습니다.
  • 회전은 설정이 변경되거나 백업 작업이 완료된 직후 트리거됩니다.
  • 잠근 버전은 최대 버전 수 계산에서 제외됩니다.

일반 참고 사항:

  • 작업 이름은 1-32자의 Unicode 문자를 포함할 수 있지만 다음 문자는 포함할 수 없습니다:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • 백업의 전체 폴더 경로의 최대 길이는 2048자입니다.
  • AFP를 통해 32000개 이상의 파일 또는 폴더를 EXT3 공유 폴더에 업로드하는 경우 이 공유 폴더를 포함한 모든 백업 작업이 실패합니다.
  • 암호화된 폴더는 마운트되거나 마운트되지 않고 백업될 수 있습니다. 그러나, 복원은 항상 마운트되지 않은 상태로 수행되며, 시작할 때 자동으로 마운트되지 않습니다. 암호화된 공유 폴더를 마운트하기 위해서는 암호화 키를 가져오거나 입력해야 합니다.
  • 암호화된 폴더를 다른 암호화된 폴더로 백업하는 것은 지원되지 않습니다.
  • 백업 소스 또는 백업 대상으로 원격 폴더를 사용하는 것은 지원되지 않습니다.
  • 백업 대상이 볼륨이면 작업 소유자만 백업 작업을 수행할 수 있습니다. 자세한 내용은 백업 대상을 참조하십시오.
  • 특정 응용 프로그램에 대한 백업이 지원됩니다. 특정 패키지가 백업을 지원하는지, 백업 내용 및 제한에 대해서 알아보려면 각 응용 프로그램의 도움말을 참조하십시오.
  • DiskStation은(는) 쓰기 가능한 EXT3/EXT4 외부 장치가 없으면 응용 프로그램 백업을 지원하지 않습니다.
  • DiskStation은(는) 쓰기 가능한 EXT3/EXT4 외부 장치가 없으면 볼륨으로의 백업을 지원하지 않습니다.
  • DiskStation은(는) EXT3/EXT4 외부 장치에서 생성된 공유 폴더만 백업할 수 있습니다.
  • 쓰기 가능한 볼륨이 없으면 시스템에서 응용 프로그램을 백업할 수 없습니다.

Windows ACL 설정 백업:

DiskStation에 있는 파일의 Windows ACL 설정 또한 대상의 파일 시스템에 따라 백업될 수 있습니다:

  • 로컬 백업 대상, 네트워크 백업 대상 및 공용 클라우드 백업 대상은 대상의 파일 시스템과 관계 없이 Windows ACL 설정을 백업할 수 있습니다.
  • 공유 폴더 동기화는 대상의 파일 시스템이 EXT4/BTRFS일 때 Windows ACL 설정을 백업할 수 있습니다.
