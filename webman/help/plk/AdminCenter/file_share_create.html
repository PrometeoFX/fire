Tworzenie folderu współdzielonego

Foldery współdzielone to podstawowe katalogi, w których można przechowywać pliki i zarządzać nimi na serwerze DiskStation. Przed zapisaniem plików należy utworzyć w systemie DSM co najmniej jeden folder współdzielony.

Aby utworzyć folder współdzielony:

  1. Ukryj ten folder współdzielony w Moje miejsca sieciowe: zapobiega wyświetlaniu folderu współdzielonego w grupie Sieć w programie Eksplorator plików Windows. Ta opcja nie wpływa na uprawnienia dostępu do folderu współdzielonego.

    Uwaga:

    Ukrywanie folderu współdzielonego w Moje miejsca sieciowe nie wpływa na prawa dostępu. Prawa dostępu do ukrytego folderu współdzielonego nie ulegną zmianie. Użytkownicy, którzy mają odpowiednie uprawnienia dostępu do folderu współdzielonego, mogą nadal uzyskać do niego dostęp przez \\nazwa serwera\nazwa folderu współdzielonego.

  2. Ukryj podfoldery i pliki przed użytkownikami bez uprawnień: po włączeniu ustawienia Ukryj podfoldery i pliki przed użytkownikami bez uprawnień użytkownicy niemający uprawnień do odczytu, którzy spróbują uzyskać dostęp do folderu współdzielonego przy użyciu protokołu Windows File Sharing, nie będą mogli wyświetlić podfolderów ani plików w tym folderze współdzielonym. Zapobiega to dezorientacji użytkowników łączących się z urządzeniami Synology, którzy widzą wiele plików i podfolderów, ale nie mogą ich otworzyć.

    Uwaga:

    • Ta funkcja jest obsługiwana wyłącznie za pośrednictwem protokołu Windows File Sharing.
    • Ustawienia uprawnień dotyczące każdego folderu i pliku można zmodyfikować w aplikacji File Station, klikając go prawym przyciskiem myszy. Z listy rozwijanej wybierz polecenie Właściwości, a następnie kliknij kartę Uprawnienie.
    • Sam folder współdzielony nie zostanie ukryty przed użytkownikami bez uprawnień po włączeniu ustawienia Ukryj podfoldery i pliki przed użytkownikami bez uprawnień. Ukryte zostaną tylko podfoldery i pliki w tym folderze współdzielonym.
  3. Włącz kosz: pliki usunięte z folderu współdzielonego są przenoszone do folderu o nazwie #recycle. Dostęp do kosza można ograniczyć do użytkowników należących do grupy administrators.
  4. Ustaw widoczność zrzutu obrazu: Dostęp do folderów i plików ze zrzutu obrazu można uzyskać za pomocą usług plików w folderach #snapshot w folderze współdzielonym. Nazwy zrzutów obrazu są tworzone na podstawie czasu GMT zrobienia zrzutu obrazu.

    Uwaga:

    • Użytkownicy nie mają prawa zapisu do kopii migawek.
    • Użytkownicy mają prawo odczytu kopii migawki tylko wtedy, gdy w czasie tworzenia migawki ich konta użytkowników były utworzone i posiadały uprawnienia do odczytu danego folderu współdzielonego.
  5. Włącz limit folderu współdzielonego: aby określić maksymalną pojemność folderów współdzielonych, zaznacz pole wyboru Włącz limit folderu współdzielonego i w polu wprowadź maksymalną pojemność (np. 10 GB).

    Uwaga:

    • Opcje folderów współdzielonych Ustaw widoczność zrzutu obrazu i Włącz limit folderu współdzielonego będą dostępne tylko, gdy w pamięci masowej używany jest system plików Btrfs.
    • Łączne użycie pamięci masowej katalogów home wszystkich użytkowników nie może przekraczać limitu katalogu homes.
  6. Włącz zaawansowaną ochronę integralności danych: Aby zapewnić ochronę folderu współdzielonego za pomocą strategii Suma kontrolna CRC32 i Kopiowanie przy zapisie, można włączyć zaawansowaną ochronę integralności danych podczas tworzenia folderu współdzielonego. Funkcja Suma kontrolna CRC32 służy do sprawdzania, czy dane są takie same, jak w momencie ich pierwotnego zapisu. Dzięki temu można wykryć wszelkie uszkodzenia danych. Funkcja Kopiowanie przy zapisie pozwala poprawić spójność danych w przypadku nieprawidłowego zamknięcia systemu.

    Uwaga:

    • Opcja jest dostępna tylko na woluminach sformatowanych w systemie plików Btrfs.
    • Można włączyć/wyłączyć tę opcję dla folderu współdzielonego podczas jego tworzenia. Nie można jej zmodyfikować po utworzeniu folderu współdzielonego.
    • Włączenie tej opcji może spowodować obniżenie wydajności systemu.
  7. Zaszyfruj ten folder współdzielony: aby zaszyfrować folder współdzielony, wykonaj następujące czynności:
    1. Określ i potwierdź nowy klucz szyfrowania. Klucz szyfrowania nie może zawierać przecinków (,) ani znaków równości (=).
    2. W razie potrzeby zaznacz pole wyboru Montuj automatycznie przy uruchamianiu, aby szyfrowany folder był montowany automatycznie podczas uruchamiania. Jeżeli opcja jest wyłączona, zaszyfrowany folder współdzielony pozostanie niezamontowany po kolejnym uruchomieniu serwera DiskStation. Aby zobaczyć niezamontowany folder współdzielony i uzyskać do niego dostęp, trzeba będzie ręcznie go zamontować, importując lub wprowadzając klucz szyfrowania.
  8. Kliknij OK, aby zakończyć.

Reguły nazewnictwa folderu współdzielonego:

  • W nazwach folderów współdzielonych nie jest istotna wielkość liter i mogą one mieć od 1 do 32 znaków. Nazwy nie mogą zawierać następujących znaków: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Następujące nazwy są zarezerwowane dla systemu: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Pierwszym znakiem nazwy folderu współdzielonego nie może być znak minus ani spacja, a ostatni znak nie może być spacją.
  • W opisie folderu współdzielonego istotna jest wielkość liter i powinien on zawierać do 64 wyświetlanych znaków Unicode.

Szyfrowanie folderów współdzielonych:

Szyfrowanie przy użyciu 256-bitowego algorytmu AES chroni dane, blokując wszystkie nieautoryzowane próby dostępu. Po zaszyfrowaniu folderu współdzielonego inni użytkownicy nie będą mogli uzyskać dostępu do zaszyfrowanych danych ani ich zmodyfikować bez klucza szyfrowania nawet po usunięciu dysków twardych z serwera DiskStation i zamontowaniu ich w innym urządzeniu.

Uwaga:

  • Foldery współdzielone można zaszyfrować tylko podczas tworzenia.
  • Nazwy plików i folderów w zaszyfrowanym folderze współdzielonym nie mogą mieć więcej niż 143 znaki łacińskie lub 43 znaki azjatyckie.
  • Zaszyfrowane foldery współdzielone nie są dostępne za pośrednictwem NFS.
  • Jeżeli zostaną przywrócone domyślne hasła przy użyciu przycisku Reset na serwerze DiskStation, zaszyfrowane foldery współdzielone zostaną odmontowane, a opcja automatycznego montowania zaszyfrowanych folderów współdzielonych zostanie wyłączona.
