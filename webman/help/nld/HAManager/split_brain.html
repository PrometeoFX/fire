Split-brain

Wanneer een high-availability cluster normaal functioneert, moet slechts een van de deel uitmakende servers de rol van actieve server op zich nemen. In dit geval detecteert de passieve server de aanwezigheid van de actieve server via de Heartbeat- en gegevensverbinding.

Worden alle Heartbeat- en dataverbindingen onderbroken dan kan het gebeuren dat beide servers proberen om de rol van actieve server over te nemen. Deze situatie wordt omschreven als een "split-brain"-fout. In dit geval worden verbindingen met het aan de high-availability cluster toegewezen IP-adres omgeleid naar een van de twee servers, wat resulteert in servicefouten.

In geval van een split-brain-fout:

  1. worden verbindingen naar het IP-adres van het cluster omgeleid naar een van de servers, wat resulteert in servicefouten. U zult problemen ondervinden bij het verbinden met een van de servers.
  2. Het systeem zal de split-brain-fout detecteren en de high-availability cluster verwijderen, waardoor beide servers worden ontkoppeld.
  3. Zodra beide servers terugkeren naar Standalone status, zult u zich kunnen aanmelden op DiskStation Manager (DSM). Op dit moment blijven alle gegevens die voor de verwijdering van de high-availability cluster werden verzameld op beide servers behouden. Na identificatie van de server met de gegevens die het meest up-to-date zijn, kunt u de servers opnieuw koppelen om opnieuw een high-availability cluster te creëren.
