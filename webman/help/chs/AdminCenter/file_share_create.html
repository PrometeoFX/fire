创建共享文件夹

共享文件夹是 DiskStation 基本的目录结构，您可在其中存储并管理文件和文件夹。存储文件前，您必须在 DSM 创建至少一个共享文件夹。

若要创建共享文件夹：

  1. 请进入控制面板 > 共享文件夹然后单击创建。
  2. 为共享文件夹指定名称和描述。
  3. 选择您想创建共享文件夹的存储器（如果只有一个存储器，请跳过此步骤）。
  4. 如需要，请修改以下选项：
    • 在“网上邻居”隐藏此共享文件夹：此选项可让该共享文件夹不在 Windows 文件资源管理器的“网络”下出现。勾选此选项并不影响共享文件夹的访问权限。

      注：

      在“网上邻居”隐藏共享文件夹并不会影响其访问权限。隐藏的共享文件夹访问权限将保持不变。拥有访问权限的用户仍可通过输入“\\服务器名称\共享文件夹名称”的方式来访问该共享文件夹。

    • 对没有权限的用户隐藏子文件夹和文件：当对没有权限的用户隐藏子文件夹和文件选项启用时，如果没有读取权限的用户试图通过 Windows 文件共享协议来访问共享文件夹，该用户将无法查看共享文件夹中的子文件夹或文件。此选项可避免用户连接到其 Synology 产品后，但无法打开大量文件和子文件夹时而感到迷惑。

      注：

      • 仅 Windows 文件共享协议支持此功能。
      • 您可右击 File Station 中文件夹和文件来修改其权限设置。从下拉菜单中选择属性，然后单击权限选项卡。
      • 启用对没有权限的用户隐藏子文件夹和文件后，对没有权限的用户不会隐藏共享文件夹本身。只有该共享文件夹中的子文件夹和文件会隐藏起来。
  5. 启用回收站：当共享文件夹中的文件被删除后，这些文件将被移动到名为 #recycle 的文件夹中。访问回收站的权限仅限属于 administrators 群组的用户。
  6. 让快照可见：快照中的文件夹和文件可通过共享文件夹下 #snapshot 文件夹中的文件服务进行访问。拍摄的快照在格林威治标准时间（GMT）后将被重命名。

    注：

    • 用户没有任何快照副本的写入权限。
    • 用户仅在拍摄快照、其用户帐户已创建并已授予共享文件夹的读取权限后才有快照副本的读取权限。
  7. 启用共享文件夹配额：要指定各共享文件夹的最大容量，请勾选启用共享文件夹配额复选框，并在以下栏中输入最大容量（如 10 GB）。

    注：

    • 只有当存储器使用 Btrfs 文件系统时，让快照可见和启用共享文件夹配额选项才能被共享文件夹使用。
    • 所有用户 home 目录的存储空间使用状况不可超出“homes”的配额。
  8. 启用高级数据完整性保护：若要用 CRC32 总和检查码和写入时复制策略保护共享文件夹，您可在创建共享文件夹时启用高级数据完整性保护。应用 CRC32 总和检查码可检查数据是否与最初写入时相同，并将在检测到任何损毁时通知用户。“写入时复制”功能有助于提高异常系统关机时的数据一致性。

    注：

    • 此选项仅适用于使用 Btrfs 文件系统的卷。
    • 您可在各共享文件夹创建时启用/禁用此选项。一旦共享文件夹创建后，此选项将无法修改。
    • 如果启用此选项，系统性能可能会受到影响。
  9. 加密此共享文件夹：如果要对共享文件夹进行加密，请执行以下操作：
    1. 指定并确认新的加密密钥。加密密钥不包括逗号（,）或等号（=）。
    2. 如需要，请勾选开机时自动装载复选框可在系统启动时自动装载加密的文件夹。如果禁用此选项，加密的共享文件夹会在下次 DiskStation 重新启动时被取消装载。若要查看并访问卸载的共享文件夹，您需要导入或输入加密密钥，藉此手动装载此文件夹。
  10. 单击确定来完成。

共享文件夹的命名规则：

  • 共享文件夹名称不分大小写，最多可以包含 1-32 个字符。但名称不得包含以下字符：! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • 以下名称保留给系统所用：.、..、global、home、homes、printers、public、satashare、surveillance、usbbackup、usbshare。
  • 共享文件夹名称的第一个字符不可为减号及空白，最后一个字符不可为空白。
  • 共享文件夹描述有区分大小写，可以是 0 到 64 个可显示的 Unicode 字符。

加密共享文件夹：

AES 256 位加密机制通过封锁所有未经授权的访问尝试来保护数据。共享文件夹加密后，那些没有加密密钥的用户将无法访问或修改加密的数据，即使将硬盘从 DiskStation 取下并在其他设备上进行装载，也是无法访问。

注：

  • 共享文件夹只可在创建时进行加密。
  • 加密的共享文件夹中的文件和文件夹名称不得超过 143 个英文字符或 43 个亚洲字符。
  • 加密的共享文件夹无法通过 NFS 进行访问。
  • 如果您使用 DiskStation 上的重置按钮来恢复默认密码，加密的共享文件夹将被卸载，而自动装载加密的文件夹选项将被禁用。
