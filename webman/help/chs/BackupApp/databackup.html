数据备份

在备份和复制的备份选项卡下，您可手动或根据自定义计划来备份 DiskStation 中的特定数据。通过此功能，系统配置、共享文件夹内容以及套件数据可以备份到本地、网络或云备份目的地。

若要创建备份任务：

  1. 请选择创建 > 数据备份任务。
  2. 按向导完成设置。请访问备份目的地以了解详情。

若要管理备份任务：

请选择一个已有的任务，然后单击下列按钮之一：

  • 编辑：修改所选任务的备份目的地、数据来源、备份计划及其它设置。
  • 删除：删除所选任务。
  • 取消：取消所选运行中的任务。
  • 立即备份：立即执行所选的备份任务。

备份循环

备份循环是一个版本管理机制。它可通过自动删除较旧的和不需要的版本并可能释放存储空间来帮助管理备份版本。您可在任务设置中启用此功能，或在创建备份任务时进行配置。

若要循环版本：

  1. 选择要对其应用版本循环的备份任务，并单击编辑。
  2. 进入任务设置，并勾选启用备份循环。
  3. 您可在此选择两种循环方法之一：
    • 从最早的版本：这是基本版本循环方法。当循环条件被触发后，最早的版本将先被删除。
    • Smart Recycle：这是智能版本循环方法。当循环条件被触发后，系统将删除保留机制所选的版本。保留机制如下：
      • 过去 24 小时，每小时的版本：系统会保留每个小时最早创建的备份版本。
      • 过去 1 天至 1 个月内，每天的版本：系统会保留每天最早创建的备份版本。
      • 较早 1 个月，每周的版本。系统会保留每周最早创建的备份版本。
  4. 指定版本数量上限。当版本数量超出此数时，版本循环将开始。该数字不包括锁定的版本。
  5. 如果您已配置重复备份计划，将显示备份版本留方式预计预览，以及基于将显示循环设置的最早还原点时间。  
  6. 单击确定来保存设置。

注：

  • 锁定的日志将永远不会被删除。请访问备份目的地以了解详情。
  • 只有多版本备份任务才会提供版本循环功能。
  • 在 DSM 5.1 或以下版本中不支持版本循环。
  • 循环将在设置更改或备份任务完成后被触发。
  • 在计算版本数量上限时，锁定的版本被排除在外。

基本注意事项：

  • 备份任务名称可为 1 到 32 个 Unicode 字符，但不可含下列字符：
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • 备份文件夹路径的长度上限为 2048 个字符。
  • 如果通过 AFP 上传到 EXT3 共享文件夹的文件或文件夹超过 32000 个，则所有备份任务（包括该共享文件夹）将会失败。
  • 装载或卸载的加密共享文件夹可进行备份。但它将始终还原为未装载并在启动时不会自动装载。系统将提示您输入或导入加密共享文件夹密钥以装载它。
  • 系统不支持将加密文件夹备份至另一个加密文件夹。
  • 系统不支持使用远程文件夹作为备份源或备份目的地。
  • 如果备份目的地为卷，则仅有任务拥有者才能执行备份任务。请访问备份目的地以了解详情。
  • 支持某些应用程序的备份。若要查看特定套件是否支持备份功能以及备份内容和限制，请参见各应用程序的帮助。
  • DiskStation 在没有可写 EXT3/EXT4 外接设备时，无法备份应用程序。
  • DiskStation 在没有可写 EXT3/EXT4 外接设备时，无法将数据备份至卷。
  • DiskStation 仅备份 EXT3/EXT4 外接设备上创建的共享文件夹。
  • 系统无法备份无可写入卷的应用程序。

备份 Windows ACL 权限设置：

根据备份目的地的文件系统不同，您也可以备份 DiskStation 文件的 Windows ACL 权限设置：

  • 本地备份目的地、网络备份目的地及公共云备份目的地可备份 Windows ACL 权限设置而无需考虑目的地的文件系统。
  • “共享文件夹同步”在目的地的文件系统为 EXT4/BTRFS 时，可以备份 Windows ACL 权限设置。
