Faça backup de seus dados no eCore Server para garantir a segurança dos dados

Visão geral

Um defeito imprevisível no servidor ou paralisação do volume representa um risco grave aos seus dados vitais. O eCore Server fornece alternativas aos usuários, local ou remotamente, para fazer backups de segurança dos seus dados mais importantes para que você esteja sempre preparado para a pior situação. Você pode escolher fazer o backup de uma ou mais maneiras para reduzir o risco de perda de dados, dependendo das exigências e situações de sua empresa.

1. Backup para um dispositivo de armazenamento externo

Ao realizar backup localmente, você pode escolher configurar o destino do backup em outra pasta no mesmo volume, grupo RAID ou até no mesmo disco físico. Nesse caso, se seus dados forem excluídos erroneamente, você conseguirá recuperá-los das cópias com backup em outra pasta. Entretanto, ao enfrentar falhas de servidor ou paralização do volume desastrosas, seus dados, bem como as cópias de backup podem ser perdidos. Portanto, sugerimos que você sempre faça backup de seus dados importantes para um dispositivo de armazenamento externo, em vez de fazê-lo no mesmo servidor. O eCore Server permite que os usuários configurem o destino do backup não apenas para unidades locais, mas também para dispositivos USB externos. Esse recurso está disponível em Menu principal > Backup e replicação. Para obter instruções passo a passo, consulte aqui.

2. Backup para outros eCore Server através da rede

O backup de seus dados para um dispositivo de armazenamento externo aumenta a disponibilidade dos dados, mas você deve pensar no risco de desastres como incêndios ou terremotos do local em que o servidor e os dispositivos de armazenamento estão localizados. O eCore Server permite que os usuários configurem um backup fora do local transferindo as cópias de seus dados para outro servidor em outra local pela rede, minimizando o risco de perda de dados devido a desastres no local do servidor. A transferência de dados do backup pode ser criptografada para proteger seus dados mais importantes com um nível maior de segurança. Com a opção de backup de nível de bloco, seu NAS fará o backup apenas dos blocos modificados e isso é especialmente prático quando o backup é executado pela Internet. Esse recurso está disponível em Menu principal > Backup e replicação. Para obter instruções passo a passo, consulte aqui.

3. Backup em um armazenamento na nuvem público

O backup remoto entre diferentes dispositivos NAS exige dois ou mais servidores, incluindo vários discos colocados dentro deles e locais para por os servidores. A situação de sua empresa pode não permitir a configuração de um backup fora do local com vários dispositivos NAS. Para proporcionar uma solução para o problema da inviabilidade de implementação do NAS para o backup de rede, o recurso de backup da Synology oferece suporte a vários serviços na nuvem públicos, o que permite backup de dados diretamente em sua conta pública na nuvem. O recurso está disponível em Backup e replicação ou Centro de pacotes dependendo de seu armazenamento na nuvem pública preferido. Para obter instruções passo a passo, consulte aqui.
