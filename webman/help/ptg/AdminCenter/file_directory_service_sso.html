Cliente SSO

Se o seu DiskStation entrou no mesmo domínio ou serviço LDAP que o servidor SSO, pode configurar o seu DiskStation como um cliente SSO. Os utilizadores podem aceder aos serviços fornecidos pelo seu DiskStation assim que iniciem a sessão no servidor SSO com as suas credenciais.

Antes de começar, certifique-se que o seu DiskStation entrou no mesmo domínio ou serviço LDAP que o servidor SSO.

Para configurar o seu DiskStation como um cliente SSO:

  1. Vá para Painel de Controlo > Domínio/LDAP > Cliente SSO.
  2. Assinale a caixa de verificação Activar o serviço SSO.
  3. Especifique o URL do servidor SSO e ID de aplicação nos campos abaixo da caixa de verificação.

    Nota:

    Para obter o ID de aplicação, escolha um dos seguintes métodos:

    • Método 1:
      1. Inicie a sessão no DSM que funciona como servidor SSO.
      2. Adicione o seu DiskStation que servirá como aplicação cliente à Lista de Aplicações no SSO Server. Para passos detalhados, consulte o artigo de Ajuda para SSO Server > Lista de Aplicações.
      3. Copie e cole o ID da aplicação alocado pelo SSO Server assim que o seu DiskStation tenha sido adicionado ao servidor SSO.
    • Método 2:
      1. Depois de introduzir o URL do servidor SSO*, clique em Registo Rápido junto do campo ID de aplicação.
      2. Na caixa de registo, introduza as credenciais de uma conta de administrador do servidor SSO nos campos Administrador de servidor e Palavra-passe.
      3. Especifique o nome da sua aplicação cliente no campo Nome da Aplicação.
      4. Certifique-se que URI de redireccionamento se refere exactamente à página de início de sessão do seu DiskStation.
      5. Clique em OK, e o seu DiskStation será adicionado ao SSO Server, com o ID de aplicação preenchido automaticamente.

    ______

    * É necessário introduzir o URL do servidor SSO idêntico (nome de anfitrião ou IP) quando configura cada cliente SSO.

  4. Clique em Aplicar para guardar as suas definições.

Nota:

  • Se o servidor SSO usar o protocolo HTTPS, o início de sessão SSO poderá ser bloqueado por navegadores caso não exista um certificado de confiança.
  • Quando o servidor SSO é acedido pelo protocolo HTTP, se o seu DiskStation (configurado com um cliente SSO) usar a ligação HTTPS, os utilizadores têm de configurar as definições dos navegadores para permitir o uso de uma ligação desprotegida.
