RAID Group*

Com DiskStation, pode combinar múltiplos discos rígidos numa única unidade de armazenamento chamada RAID Group. Podem ser criados volumes ou iSCSI LUNs (Nível do Bloco) por cima de RAID Groups. Os volumes ou iSCSI LUNs podem ser expandidos se um grupo RAID tiver espaço que possa ser atribuído. Os diferentes tipos de RAID oferecem vários níveis de proteção de dados. Para uma lista geral dos tipos de RAID suportados por DiskStation, consulte Geral.

Para criar um grupo RAID:

  1. Clique no botão Criar.
  2. Escolha uma das seguintes opções:
    • Grupo RAID para volume único ou iSCSI LUN (Nível do Bloco): Oferece melhor desempenho.
    • Grupo RAID para múltiplos volumes ou iSCSI LUNs (Nível do Bloco): Oferece melhor flexibilidade de gestão.
  3. Siga o assistente para concluir o processo.

Nota:

  • Um grupo RAID deve incluir apenas discos rígidos do mesmo tipo de unidade rígida.
  • Um grupo RAID para múltiplos volumes ou iSCSI LUNs (Nível do Bloco) que usa RAID 5 ou RAID 6 pode conter múltiplos Conjuntos RAID. Cada Conjunto RAID pertencendo ao mesmo grupo RAID serão do mesmo tipo de RAID (por exemplo, RAID 5 ou RAID 6). O número máximo de discos para cada Conjunto RAID é 12. Quando um Conjunto RAID receber 12 discos, o disco seguinte atribuído ao grupo RAID poderá ser atribuído a um novo Conjunto RAID. O novo Conjunto RAID deve conter o número mínimo de discos necessários para esse tipo de RAID específicos (ou seja, três discos para RAID 5 e quatro discos para RAID 6).
  • Se pretender criar um Grupo RAID com um determinado disco rígido mas este não aparece no Assistente de Criação de Grupo RAID, verifique o estado do disco rígido em Gestão de HDD.

Para remover um grupo RAID:

  1. Selecione o grupo RAID que pretende remover.
  2. Clique no botão Remover e siga o assistente para concluir o processo.

Nota: os volumes e iSCSI LUNs no Grupo RAID serão eliminados ao remover o Grupo RAID. Certifique-se de que não elimina dados importantes.

Para reparar um grupo RAID:

Um grupo RAID cujo estado é Degradado pode ser reparado, se o grupo RAID for um dos seguintes tipos: RAID 1, RAID 5, RAID 6 e RAID 10. Os dados existentes no grupo RAID não serão eliminados.

Siga os passos abaixo para repara o seu grupo RAID:

  1. Desligue o seu DiskStation. (Pode ignorar este passo se o seu modelo suportar hot-swapping.)
  2. Substitua o disco rígido com falha como indicado na informação do disco, localizada no estado do grupo RAID.
  3. Clique em Gerir.
  4. Seleccione Reparar.
  5. Siga o assistente para concluir o processo.

Nota:

  • O tamanho do disco rígido de substituição deve ser igual ou maior que o tamanho do disco mais pequeno no Grupo RAID.
  • O estado do disco rígido a utilizar deve ser "Inicializado" ou "Não Inicializado".

Para alterar o tipo de RAID de um grupo RAID

Para alterar o tipo de RAID de um RAID Group, deverá haver discos rígidos disponíveis instalados no seu DiskStation. Atualmente, são suportados os seguintes tipos de alterações a RAID:

  • Basic para RAID 1 ou RAID 5
  • RAID 1 para RAID 5
  • RAID 5 para RAID 6
  • Adicionar disco espelhado ao RAID 1

Siga os passos abaixo para alterar o tipo RAID do seu Grupo RAID:

  1. Selecione o grupo RAID que pretende alterar.
  2. Clique em Gerir.
  3. Selecione Alterar tipo RAID.
  4. Siga o assistente para concluir o processo.

Nota:

  • O estado de um RAID Group deve ser Normal antes de poder alterar o seu tipo de RAID. Caso contrário, ocorrerá a perda de dados. Se o estado do Grupo RAID for Degradado repare-o primeiro.
  • Para alterar o tipo de RAID de um RAID Group, deverão estar instalados discos rígidos suficientes no seu DiskStation e o tamanho do novo disco deve ser maior do que o disco mais pequeno do RAID Group.
  • O estado do novo disco deve ser "Inicializado" ou "Não Inicializado".
  • O tipo de RAID de um grupo RAID com múltiplos conjuntos RAID não pode ser alterado.

Para expandir o tamanho de um Grupo RAID

Pode expandir o tamanho de um Grupo RAID substituindo discos rígidos existentes por uns de tamanho superior ou adicionando discos rígidos.

Nota:

  • Para expandir um RAID Group com discos de 3 TB ou maiores, certifique-se que o seu RAID Group foi criado com DSM 3.0 ou posterior.

Para expandir um Grupo RAID substituindo discos rígidos existentes por outros de tamanho maior:

Pode expandir o tamanho de um grupo RAID substituindo discos rígidos existentes por uns de tamanho superior nos seguintes tipos de RAID: RAID 1, RAID 5 e RAID 6. Volumes associados ou iSCSI LUN no grupo RAID para um único volume ou iSCSI LUN serão automaticamente expandidos. Siga os passos em baixo:

  1. Desligue o seu DiskStation. Pode ignorar este passo se o seu modelo suportar hot-swapping.
  2. Substitua um disco rígido por outro de tamanho superior.
  3. Ligue o DiskStation e repare o RAID Group.
  4. Repita os passos 1 a 3 até que todos os discos rígidos tenham sido substituídos por discos rígidos maiores.
  5. O sistema irá expandir o grupo RAID automaticamente.

Nota:

  • Todos os serviços do Gestor de Armazenamento serão interrompidos durante o processo de expansão.
  • Apenas pode expandir o disco rígido quando o estado do Grupo RAID for Normal; de outra forma irá resultar na perda de dados. Se o estado do Grupo RAID for Degradado repare-o primeiro.

Para expandir um Grupo RAID adicionando discos rígidos:

Esta função aplica-se aos tipos de RAID JBOD, RAID 5 e RAID 6. Para isso:

  1. Seleccione o RAID Group que pretende expandir.
  2. Clique em Gerir.
  3. Selecione Adicionar Disco.
  4. Siga o assistente para concluir a configuração.

Nota:

  • O status do Grupo RAID deve ser Normal.
  • Para os tipos de Grupo RAID RAID 5 e RAID 6, o tamanho do disco rígido deve ser maior do que o disco mais pequeno no Grupo RAID.
  • O estado do disco rígido seleccionado deve ser "Inicializado" ou "Não Inicializado".

Para editar a descrição de um grupo RAID:

  1. Selecione o grupo RAID que pretende editar
  2. Clique em Gerir.
  3. Selecione Editar descrição.
  4. Edite a descrição e clique em OK para guardar os resultados.

Análise de dados

A análise de dados é uma função de manutenção de dados que inspeciona RAID Groups e corrige quaisquer inconsistências de dados detetadas. Esta função pode ser usada com Grupos RAID que implementem RAID 5 ou RAID 6.

Recomendamos a realização de uma análise de dados regular, para manter a consistência de dados e evitar perdas de dados críticos caso o disco falhe.

Para realizar a análise de dados:

  1. Seleccione o Grupo RAID no qual realizar a análise de dados.
  2. Clique em Gerir
  3. Seleccione Análise de dados.

Nota:

  • A análise de dados apenas pode ser realizada quando o estado do Grupo RAID for Normal.
  • Não desligue o sistema antes de terminar a análise de dados. Se o sistema for encerrado antes da conclusão, a análise de dados deverá ser reiniciada.
