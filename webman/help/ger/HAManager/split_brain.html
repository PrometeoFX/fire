Split-Brain-Fehler

Wenn ein High-availability-Cluster normal funktioniert, sollte nur einer der Mitgliedsserver die Rolle des aktiven Servers übernehmen. In diesem Fall erkennt der passive Server die Gegenwart des aktiven Servers sowohl über die Heartbeat- als auch über die Datenverbindung.

Wenn alle Heartbeat- und Datenverbindungen verloren gehen, könnten beide Server versuchen, die Rolle des aktiven Servers zu übernehmen. Diese Situation wird als „Split-brain“-Fehler bezeichnet. In diesem Fall werden Verbindungen an die mit dem High-availability-Cluster verbundene IP-Adresse auf einen der beiden Server zurückgeleitet, was zu Fehlern bei den Diensten führt.

Falls ein Split-Brain Fehler auftritt:

  1. Verbindungen zur Cluster-IP-Adresse werden an einen der beiden Server zurückgeleitet, was zu Fehlern bei den Diensten führt. Sie werden Schwierigkeiten haben, die Verbindung zu den Servern herzustellen.
  2. Das System wird den Split-Brain-Fehler erkennen und den High-availability-Cluster entfernen und die beiden Server trennen.
  3. Wenn beide Server zum Status Einzeln zurückgekehrt sind, können Sie sich im DiskStation Manager (DSM) anmelden. Zu diesem Zeitpunkt werden Daten, die vor dem Entfernen des High-availability-Clusters gesammelt wurden, auf beiden Servern bleiben. Nach der Erfassung, welcher der beiden Server die aktuellsten Daten enthält, können Sie die beiden Server wieder einbinden, um wieder einen High-availability-Cluster zu erstellen.
