Sicherung Ihrer Daten auf ein Synology NAS, um die Datensicherheit zu gewährleisten

Übersicht

Unvorhergesehene Fehlfunktion des Servers oder ein Absturz des Volume stellen ein großes Risiko für Ihre wichtigsten Daten dar. Das Synology NAS bietet Benutzern die Alternative zur lokalen oder remote Sicherung von Kopien der wichtigsten Daten, so dass Sie immer auf das Schlimmste vorbereitet sind. Sie können sich je nach Anforderungen und Situationen Ihres Unternehmens für eine Sicherung mit einer oder mehreren Methoden entscheiden.

1. Sicherung auf einen externen Speicher

Wenn Sie eine lokale Sicherung vornehmen, können Sie als Sicherungsziel einen anderen Ordner auf dem selben Volume, RAID Gruppe oder sogar derselben Festplatte auswählen. In diesem Fall können Sie die Daten nach einem versehentlichen Löschen aus den in einem anderen Ordner gesicherten Kopien wiederherstellen. Wenn allerdings der Server ausfällt oder das Volume abstürzt, könnten Ihre Daten mitsamt den gesicherten Kopien verloren gehen. Wir empfehlen daher immer eine Sicherung Ihrer wichtigsten Daten auf ein externes Speichergerät als auf demselben Server. Das Synology NAS ermöglicht Ihnen als Einstellung des Sicherungsziels nicht nur lokale Laufwerke, sondern auch externe USB-Geräte. Die Funktion ist unter Hauptmenü > Datensicherung & Replikation verfügbar. Eine Schritt-für-Schritt Anleitung finden Sie hier.

2. Auf ein anderes Synology NAS über das Netzwerk sichern

Die Sicherung Ihrer Daten auf ein externes Speichergerät verbessert die Datenverfügbarkeit, aber denken Sie an die Risiken von Naturkatastrophen wie z.B. Feuer oder Erdbeben an dem Ort, an dem sich Ihr Server und Ihre Speichergeräte befinden. Das Synology NAS ermöglicht Benutzern die Einrichtung einer Sicherung an einem anderen Ort (Off-Site), indem Ihre Daten über das Netzwerk auf einen anderen Server übertragen werden, was das Risiko eines Datenverlusts wegen einer Katastrophe am Standort des Servers minimiert. Die Datenübertragung der Sicherung kann verschlüsselt werden, um Ihre wichtigen Daten mit einer höheren Sicherheitsstufe zu schützen. Mit der Sicherungsoption auf Blockebene wird das NAS nur geänderte Blöcke sichern. Dies ist besonders dann praktisch, wenn die Sicherung über das Internet ausgeführt wird. Die Funktion ist unter Hauptmenü > Datensicherung & Replikation verfügbar. Eine Schritt-für-Schritt Anleitung finden Sie hier.

3. Sicherung auf einem öffentlichen Cloudspeicher

Eine Remote Sicherung unter verschiedenen NAS Geräten erfordert zwei oder mehr Server, sowie eine entsprechende Anzahl an Festplatten, und den Platz für die Aufstellung der Server. Die Situation in Ihrem Unternehmen könnte die Einrichtung einer Off-Site Sicherung mit mehreren NAS Geräten möglicherweise nicht zulassen. Um das Problem der fehlenden NAS Umgebung für eine Netzwerksicherung zu lösen, bietet die Synology Sicherungsfunktion die Unterstützung für verschiedene öffentliche Clouddienste, um Ihre Daten direkt in Ihrem öffentlichen Cloud-Konto zu sichern. Diese Funktion ist je nach Ihrem bevorzugten öffentlichen Cloudspeicher unter Datensicherung & Replikation oder Paketzentrum verfügbar. Eine Schritt-für-Schritt Anleitung finden Sie hier.
