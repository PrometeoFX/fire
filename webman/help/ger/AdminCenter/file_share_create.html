Freigegebenen Ordner erstellen

Gemeinsame Ordner sind die Grundverzeichnisse, in denen Sie Dateien und Ordner auf Ihrem DiskStation speichern und verwalten können. Bevor Sie in DSM Dateien speichern können, muss mindestens ein gemeinsamer Ordner erstellt werden.

Freigegebenen Ordner erstellen:

  1. Gehen Sie zu Systemsteuerung > Gemeinsamer Ordner und klicken Sie auf Erstellen.
  2. Geben Sie einen Namen und eine Beschreibung für den gemeinsamen Ordner an.
  3. Wählen Sie einen Speicher, auf dem der gemeinsame Ordner erstellt werden soll (überspringen Sie diesen Schritt, wenn nur ein Speicher vorhanden ist).
  4. Ändern Sie die folgenden Optionen, falls erforderlich:
    • Diesen gemeinsamen Ordner unter „Netzwerkumgebung“ ausblenden: Verhindert, dass der gemeinsame Ordner unter „Netzwerk“ in Windows Explorer angezeigt wird. Diese Option beeinflusst nicht die Berechtigungen für den Zugriff auf den gemeinsamen Ordner.

      Hinweis:

      Das Verbergen eines freigegebenen Ordners in "Netzwerkumgebung" hat keine Auswirkungen auf dessen Zugriffsrechte. Die Zugriffsrechte für den ausgeblendeten gemeinsamen Ordner bleiben unverändert. Benutzer mit den richtigen Rechten für den Zugriff auf den freigegebenen Ordner können immer noch auf diesen zugreifen, indem sie „\\server name\name des freigegebenen Ordners“ eingeben.

    • Unterordner und Dateien vor Benutzern ohne Berechtigungen ausblenden: Wenn Unterordner und Dateien vor Benutzern ohne Berechtigungen ausblenden aktiviert ist und ein Benutzer ohne Leseberechtigungen versucht, über das Windows-Dateifreigabeprotokoll auf einen gemeinsamen Ordner zuzugreifen, kann der Benutzer keine Unterordner oder Dateien im gemeinsamen Ordner anzeigen. Dies vermeidet Verwirrung bei den Benutzern, wenn sie die Verbindung zum Synology-Produkt herstellen und auf eine große Anzahl von Dateien und Unterordnern treffen, die sie nicht öffnen können.

      Hinweis:

      • Diese Funktion wird nur mit dem Windows-Dateifreigabeprotokoll unterstützt.
      • Sie können die Privilegieneinstellungen jedes Ordners und jeder Datei in File Station ändern, indem Sie mit der rechten Maustaste auf den Ordner oder die Datei klicken. Wählen Sie Eigenschaften aus dem Dropdown-Menü aus und klicken Sie auf die Registerkarte Berechtigung.
      • Der gemeinsame Ordner wird für Benutzer ohne Berechtigungen nicht ausgeblendet, wenn Unterordner und Dateien vor Benutzern ohne Berechtigungen ausblenden aktiviert ist. Es werden nur die Unterordner und Dateien in diesem gemeinsamen Ordner ausgeblendet.
  5. Papierkorb aktivieren: Wenn Dateien im freigegebenen Ordner gelöscht werden, werden sie in einen Ordner namens #recycle verschoben. Der Zugriff auf den Papierkorb kann auf Benutzer der Gruppe administrators beschränkt werden.
  6. Schnappschuss sichtbar machen: Auf Ordner und Dateien in Schnappschüssen kann über Dateidienste in #snapshot-Ordnern des gemeinsamen Ordners zugegriffen werden. Schnappschüsse werden nach der Greenwich Mean Time (GMT) benannt, wenn sie aufgenommen werden.

    Hinweis:

    • Benutzer dürfen in Schnappschusskopien nicht schreiben.
    • Benutzer dürfen nur in der Schnappschusskopie schreiben, wenn ihr Benutzerkonto zum Zeitpunkt der Erstellung des Schnappschusses bereits erstellt war und Ihnen die Berechtigung zum Lesen des freigegebenen Ordners gewährt wurde.
  7. Gemeinsames Ordnerkontingent aktivieren: Um die maximale Kapazität für jeden gemeinsamen Ordner anzugeben, aktivieren Sie das Kontrollkästchen Gemeinsames Ordnerkontingent aktivieren und geben die maximale Kapazität (z. B. 10 GB) in das folgende Feld ein.

    Hinweis:

    • Nur wenn der Speicher für das Btrfs-Dateisystem konfiguriert ist, sind die Optionen Schnappschuss sichtbar machen und Freigegebenes Ordnerkontingent aktivieren für freigegebene Ordner verfügbar.
    • Die kombinierte Speichernutzung der home-Verzeichnisse aller Benutzer kann das Kontingent von "homes" nicht überschreiten.
  8. Erweiterten Datenintegritätsschutz aktivieren: Um einen gemeinsamen Ordner mit den Strategien CRC32 checksum und Kopieren-nach-Schreiben zu schützen, können Sie bei der Erstellung des gemeinsamen Ordners den erweiterten Datenintegritätsschutz aktivieren. Mit CRC32 checksum wird geprüft, ob die Daten sich noch in ihrem ursprünglichen Zustand befinden. Wird eine Beschädigung festgestellt, wird der Benutzer benachrichtigt. Kopieren-nach-Schreiben verbessert die Datenkonsistenz, nachdem ein System unnormal heruntergefahren wurde.

    Hinweis:

    • Diese Option ist nur für Volumes mit dem Btrfs-Dateisystem verfügbar.
    • Diese Option können Sie für jeden gemeinsamen Ordner während der Erstellung aktivieren bzw. deaktivieren. Nach der Erstellung des gemeinsamen Ordners kann diese Option nicht mehr geändert werden.
    • Die Aktivierung dieser Option kann die Systemleistung beeinträchtigen.
  9. Diesen gemeinsamen Ordner verschlüsseln: Zum Verschlüsseln des gemeinsamen Ordners gehen Sie wie folgt vor:
    1. Geben Sie einen neuen Verschlüsselungsschlüssel an und bestätigen Sie ihn. Der Verschlüsselungsschlüssel darf keine Kommas (,) oder Gleichheitszeichen (=) enthalten.
    2. Soweit erforderlich, aktivieren Sie das Kästchen Beim Start automatisch anhängen, um den verschlüsselten Ordner automatisch anzuhängen, wenn das System gestartet wird. Wenn diese Option nicht aktiviert ist, wird der verschlüsselte Ordner beim nächsten Start von DiskStation getrennt. Um einen getrennten gemeinsamen Ordner anzuzeigen oder darauf zuzugreifen, müssen Sie den Ordner manuell anhängen, indem Sie den Schlüssel für die Verschlüsselung importieren oder eingeben.
  10. Klicken Sie zum Beenden auf OK.

Namensregeln für gemeinsame Ordner:

  • Bei Namen für gemeinsame Ordner wird die Groß-/Kleinschreibung nicht berücksichtigt und sie können 1-32 Zeichen umfassen. Der Name darf die folgenden Zeichen nicht enthalten: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Die folgenden Namen sind für die Systemnutzung reserviert: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Das erste Zeichen eines gemeinsamen Ordners darf kein Minus- oder Leerzeichen sein und das letzte Zeichen darf ebenfalls kein Leerzeichen sein.
  • Bei der Beschreibung des gemeinsam genutzten Ordners wird die Groß-/Kleinschreibung berücksichtigt und sie ist auf bis zu 64 anzeigbare Unicode-Zeichen beschränkt.

Gemeinsame Ordner verschlüsseln:

Die AES 256-Bit-Verschlüsselung kann Daten vor nicht autorisierten Zugriffsversuchen schützen. Wenn ein gemeinsamer Ordner verschlüsselt ist, kann niemand ohne Schlüssel auf die verschlüsselten Daten zugreifen oder sie ändern, selbst wenn sie die Festplatten von DiskStation entfernen und sie in einem anderen Gerät installieren.

Hinweis:

  • Gemeinsame Ordner können nur bei ihrer Erstellung verschlüsselt werden.
  • Die Namen der Dateien und Ordner in verschlüsselten gemeinsamen Ordnern dürfen 143 englische oder 43 asiatische Zeichen nicht überschreiten.
  • Die verschlüsselten gemeinsamen Ordner können nicht über NFS aufgerufen werden.
  • Wenn Standard-Kennwörter mit der Reset-Taste des DiskStation wiederhergestellt werden, werden verschlüsselte gemeinsame Ordner deaktiviert und die Option, verschlüsselte gemeinsame Ordner automatisch anzuhängen, wird deaktiviert.
