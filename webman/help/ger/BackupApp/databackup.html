Datensicherung

Auf der Registerkarte Datensicherung von Datensicherung & Replikation können bestimmte Daten auf dem DiskStation manuell oder nach einem benutzerdefinierten Zeitplan gesichert werden. Mit dieser Funktion können Systemkonfigurationen, der Inhalt der freigegebenen Ordner und Paketdaten mit einem lokalen, Netzwerk- oder Cloud-Datensicherungsziel gesichert werden.

Sicherungsaufgabe erstellen:

  1. Wählen Sie Erstellen > Datensicherungsaufgabe aus.
  2. Folgen Sie dem Assistenten, um das Setup abzuschließen. Weitere Informationen finden Sie unter Datensicherungsziel.

Datensicherungsaufgaben verwalten:

Wählen Sie eine vorhandene Aufgabe aus und klicken Sie auf eine der folgenden Schaltflächen:

  • Bearbeiten: Ändert Ziel, Quelle, Zeitplan und andere Einstellungen der gewählten Aufgabe.
  • Löschen: Löscht die ausgewählte Aufgabe.
  • Abbrechen: Beendet die ausgewählte, derzeit ausgeführte Aufgabe.
  • Jetzt Datensicherung durchführen: Führt sofort eine Sicherung der ausgewählten Aufgabe durch.

Sicherungsrotation

Bei der Sicherungsrotation handelt es sich um einen Versionsmanagementmechanismus. Er unterstützt die Verwaltung von Sicherungsversionen durch das automatische Löschen älterer, nicht mehr benötigter Versionen und die potenzielle Freigabe von Speicherplatz. Sie können diese Funktion unter Aufgabeneinstellungen aktivieren oder bei der Erstellung einer Datensicherungsaufgabe konfigurieren.

Versionen rotieren:

  1. Wählen Sie die Datensicherungsaufgabe aus, auf die Sie die Versionsrotation anwenden möchten, und klicken Sie auf Bearbeiten.
  2. Gehen Sie zur Registerkarte Aufgabeneinstellungen und aktivieren Sie Sicherungsrotation aktivieren.
  3. Hier können Sie aus zwei Rotationsmethoden auswählen:
    • Von den frühesten Versionen: Dies ist eine Standard-Versionsrotationsmethode. Wenn die Rotationsbedingung ausgelöst wird, wird die früheste Version zuerst entfernt.
    • Smart Recycle: Dies ist eine intelligente Versionsrotationsmethode. Wenn die Rotationsbedingung ausgelöst wird, entfernt das System die vom Aufbewahrungsmechanismus ausgewählten Versionen. Der Aufbewahrungsmechanismus ist wie folgt:
      • Stündliche Versionen der letzten 24 Stunden: Das System bewahrt die früheste in jeder Stunde erstellte Version auf.
      • Tägliche Versionen vom vergangenen 1 Tag bis zu 1 Monat: Das System bewahrt die früheste an jedem Tag erstellte Version auf.
      • Wöchentliche Version älter als 1 Monat: Das System bewahrt die früheste in jeder Woche erstellte Version auf.
  4. Legen Sie die maximale Anzahl der Versionen fest. Wenn die Anzahl der Versionen diesen Wert überschreitet, beginnt die Versionsrotation. Die Anzahl umfasst keine gesperrten Versionen.
  5. Wenn Sie einen wiederholten Datensicherungszeitplan konfiguriert haben, werden die Schätzung in Bezug auf die Aufbewahrung Ihrer Sicherungsversionen und der auf Ihren Rotationseinstellungen beruhende früheste Wiederherstellungszeitpunkt in der Vorschau angezeigt.   
  6. Klicken Sie auf OK, um Ihre Einstellungen zu speichern.

Hinweise:

  • Gesperrte Versionen werden nicht entfernt. Weitere Informationen finden Sie unter Datensicherungsziel.
  • Nur bei Sicherungsaufgaben für mehrere Versionen sind die Versionsrotationsfunktionen verfügbar.
  • In DSM 5.1 oder niedriger wird die Versionsrotation nicht unterstützt.
  • Die Rotation wird direkt nach Änderung der Einstellung oder nach Abschluss der Sicherungsaufgabe ausgelöst.
  • Bei der Berechnung der maximalen Anzahl von Versionen werden gesperrte Versionen ignoriert.

Allgemeine Hinweise:

  • Die Namen von Aufgaben können 1 bis 32 Unicode-Zeichen enthalten, aber nicht die folgenden Zeichen:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • Die maximale Länge des kompletten Ordnerpfads für die Sicherung beträgt 2048 Zeichen.
  • Wenn über AFP mehr als 32.000 Dateien oder Ordner in einen freigegebenen EXT3-Ordner hochgeladen werden, schlagen alle Datensicherungsaufgaben mit dem freigegebenen Ordner fehl.
  • Ein verschlüsselter Ordner kann sowohl im angehängten als auch im getrennten Zustand gesichert werden. Jedoch wird er immer als nicht angehängt wiederhergestellt und beim Starten nicht automatisch angehängt. Sie müssen den Verschlüsselungsschlüssel für den verschlüsselten freigegebenen Ordner eingeben oder importieren, um ihn anzuhängen.
  • Die Sicherung eines verschlüsselten Ordners in einen anderen verschlüsselten Ordner wird nicht unterstützt.
  • Die Verwendung eines Remote-Ordners als Datensicherungsquelle oder -ziel wird nicht unterstützt.
  • Wenn das Datensicherungsziel ein Volume ist, kann nur der Aufgabenbesitzer die Datensicherungsaufgabe ausführen. Weitere Informationen finden Sie unter Datensicherungsziel.
  • Die Sicherung für bestimmte Anwendungen wird unterstützt. In der Hilfe jeder Anwendung erhalten Sie Informationen, ob ein bestimmtes Paket die Datensicherung unterstützt, sowie zum Datensicherungsinhalt und deren Einschränkungen.
  • Der DiskStation unterstützt nicht die Anwendungssicherung ohne ein beschreibbares externes EXT3/EXT4-Gerät.
  • Der DiskStation unterstützt nicht die Sicherung in ein Volume ohne ein beschreibbares externes EXT3/EXT4-Gerät.
  • DiskStation kann nur freigegebene Ordner sichern, die auf externen EXT3/EXT4-Geräten erstellt wurden.
  • Ohne beschreibbares Volume kann das System keine Anwendungen sichern.

Sichern der Windows ACL-Einstellungen:

Die Windows ACL-Einstellungen der Dateien auf DiskStation können je nach Dateisystem des Ziels ebenfalls gesichert werden:

  • Windows ACL-Einstellungen können, unabhängig vom Dateisystem des Ziels, mit Lokales Datensicherungsziel, Netzwerksicherungsziel und Datensicherungsziel öffentliche Cloud gesichert werden
  • Synchronisierung freigegebener Ordner kann die Windows ACL-Einstellungen sichern, wenn das Dateisystem des Ziels EXT4/BTRFS ist.
