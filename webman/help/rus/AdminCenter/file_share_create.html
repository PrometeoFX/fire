Создание папки общего доступа

Папки общего доступа являются базовыми каталогами, где можно хранить и управлять файлами и папками на устройстве DiskStation. Перед хранением файлов необходимо создать минимум одну общую папку в DSM.

Создание папки общего доступа:

  1. Скрыть эту папку общего доступа в меню "Сетевое окружение". Папка общего доступа не будет отображаться в меню "Сеть" в Проводнике Windows. Данный параметр не влияет на права доступа к папке общего доступа.

    Примечание.

    Скрытие папки общего доступа в меню "Сетевое окружение" не влияет на права доступа к ней. Право доступа к скрытой папке общего доступа останется неизменным. Пользователи, имеющие необходимые права доступа к папке общего доступа, смогут получать доступ к папке посредством ввода "\\имя сервера\имя папки общего доступа".

  2. Скрыть подпапки и файлы от пользователей без прав доступа: При активации пункта Скрыть подпапки и файлы от пользователей без прав доступа пользователь, выполняющий доступ к общей папке по протоколу обмена файлами Windows, не сможет просматривать подпапки или файлы в общей папке, если у него нет прав чтения. Это позволяет пользователям при подключении к продуктам Synology не теряться при наличии большого числа папок и файлов, к которым у них нет доступа.

    Примечание.

    • Данная функция поддерживается только в протоколе обмена файлами Windows.
    • Вы можете настроить параметры прав для каждой папки и файла в File Station, нажав его правой кнопкой мыши. Выберите Свойства в раскрывающемся меню и нажмите вкладку Разрешение.
    • Сама общая папка не будет скрыта от пользователей без прав доступа при включении параметра Скрыть подпапки и файлы от пользователей без прав доступа. Будут скрыты только подпапки и файлы в данной общей папке.
  3. Включить корзину. При удалении файлов из папки общего доступа они будут перемещены в папку #recycle. Доступ к корзине можно ограничить пользователями, принадлежащими группе administrators.
  4. Сделать мгновенный снимок видимым. К папкам и файлам в моментальных снимках можно получить доступ с помощью файловых служб в папках #snapshot в общей папке. Моментальным снимкам присваиваются имена, отображающие время по Гринвичу, когда они были сделаны.

    Примечание.

    • Пользователи не имеют права на запись для любой моментальной копии.
    • Пользователи имеют право на чтение моментальной копии только, если во время создания моментального снимка их учетные записи были созданы и им были назначены права на чтение папки общего доступа.
  5. Включить квоту общей папки. Чтобы указать максимальную емкость каждой папки общего доступа, установите флажок в поле Включить квоту папки общего доступа и введите максимальную емкость (например, 10 ГБ) в соседнее поле.

    Примечание.

    • Параметры Сделать мгновенный снимок видимым и Включить квоту папки общего доступа будут доступны для папок общего доступа, только если хранилище использует файловую систему BTRFS.
    • Использование объединенного хранилища домашних каталогов всех пользователей не должно превышать квоту папок homes.
  6. Включить усиленную защиту подлинности данных. Чтобы защитить общую папку с помощью контрольной суммы CRC32 и метода копирования при записи, при создании общей папки можно включить усиленную защиту подлинности данных. С помощью метода контрольной суммы CRC32 можно проверить идентичность данных исходным данным, выявить любые несоответствия между ними и получить соответствующее уведомление. Копирование при записи обеспечивает согласованность данных в случае внезапного сбоя системы.

    Примечание.

    • Данной функцией можно воспользоваться только в томах с файловой системой BTRFS.
    • Можно включить/отключить данную функцию при создании каждой папки общего доступа. После создания папки общего доступа изменить значение данного параметра нельзя.
    • Активация данной функции может влиять на производительность системы.
  7. Зашифровать папку общего доступа. Чтобы зашифровать папку общего доступа, выполните следующие действия.
    1. Укажите и подтвердите новый ключ шифрования. Ключ шифрования не может содержать запятые (,) и знаки равенства (=).
    2. При необходимости установите флажок Подключать автоматически при запуске, чтобы автоматически подключать зашифрованную папку при запуске системы. Если данный параметр отключен, зашифрованная папка общего доступа будет отключена при следующем запуске DiskStation. Для просмотра или доступа к удаленной папке общего доступа необходимо подключить ее вручную, выполнив импорт или введя ключ шифрования.
  8. Нажмите OK для завершения.

Правила наименования папок общего доступа.

  • Названия общих папок вводятся с учетом регистра и могут быть длиной от 1 до 32 символов. Имена не могут содержать следующие символы: ! " # $ % & ' ( ) * + , / : ; <= > ? @ [ ] \ ^ ` { } | ~
  • Следующие имена зарезервированы для системного использования: ., .., global, home, homes, printers, public, satashare, surveillance, usbbackup, usbshare.
  • Первый символ в названии общей папки не может быть знаком минуса или пробелом, последний символ также не может быть пробелом.
  • Общая папка description чувствительна к регистру и может содержать до 64 отображаемых символов в кодировке Unicode.

Шифрование общих папок:

256-битное шифрование AES блокирует все попытки несанкционированного доступа к данным. После шифрования папки общего доступа без ключа шифрования другие пользователи не смогут использовать зашифрованные данные даже при удалении жестких дисков из устройства DiskStation и их установки на собственное устройство.

Примечание.

  • Общие папки можно зашифровать и во время создания.
  • Названия файлов и папок в зашифрованных общих папках не могут превышать 143 латинские буквы или 43 азиатских знака.
  • К зашифрованным папкам общего доступа нельзя получить доступ по протоколу NFS.
  • Если вы восстанавливаете пароли по умолчанию с помощью кнопки сброса, расположенной на устройстве DiskStation, зашифрованные папки общего доступа будут отключены, а параметр автоматического подключения зашифрованных папок общего доступа будет деактивирован.
