Чтобы гарантировать безопасность данных, архивируйте их на eCore Server

Обзор

Непредвиденный сбой на сервере или отказ тома могут обусловить серьезные риски для ваших важных данным. eCore Server предоставляет альтернативы для безопасного хранения — как локального, так и удаленного — копий самых важных данных, чтобы вы всегда были готовы к наихудшим сценариям. В зависимости от требований и ситуаций в бизнесе, для уменьшения риска потери данных имеется возможность архивировать одним или несколькими способами.

1. Архивация на внешнее запоминающее устройство

При локальной архивации можно выбрать в качестве места архивации другую папку на том же томе, группе RAID или даже на том же физическом диске. В этом случае, если данные будут по ошибке удалены, вы сможете восстановить их из архивных копий в другой папке. Однако в случае сбоев сервера или отказа тома данные, а также архивные копии могут быть потеряны. Поэтому рекомендуется всегда архивировать важные данные не на тот же сервер, а на внешнее запоминающее устройство. eCore Server позволяет задавать в качестве места архивации не только локальные диски, но также и внешние устройства USB. Эта возможность доступна на странице Главное меню > Резервирование и репликация. На странице приведены пошаговые инструкции.

2. Архивация на другое устройство eCore Server по сети

Архивация данных на внешнее запоминающее устройство улучшает доступность данных, но не стоит забывать о риске стихийных бедствий, таких как пожар или землетрясения, в местах расположения сервера и запоминающих устройств. eCore Server позволяет пользователям настроить архивацию за пределами текущей вычислительной среды путем передачи копий данных по сети на другой сервер, расположенный в другом месте. Это сводит к минимуму риск потери данных в случае стихийных бедствий в месте расположения сервера. Для обеспечения более высокого уровня безопасности важных данных передача данных резервных копий может быть зашифрована. При использовании функции архивации на уровне блоков NAS будет архивировать только измененные блоки. Это особенно удобно при выполнении архивации через Интернет. Эта возможность доступна на странице Главное меню > Резервирование и репликация. На странице приведены пошаговые инструкции.

3. Архивация в публичное облако

Удаленная архивация между разными NAS-устройствами требует наличия двух или больше серверов, включая множество дисков на них, и мест расположения серверов. Ситуация в организации может не позволять настроить архивацию вне вычислительной среды с несколькими NAS-устройствами. Для решения проблемы рисков развертывания NAS для сетевой архивации функция архивации Synology обеспечивает несколько публичных облачных служб, с помощью которых можно архивировать данные непосредственно в учетную запись публичного облака. Эта функция доступна на странице Резервирование и репликация или Центр пакетов, в зависимости от выбранного хранилища в публичном облаке. На странице приведены пошаговые инструкции.
