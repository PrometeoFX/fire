Резервирование данных

На вкладке Резервирование данных в меню Резервирование данных и репликация можно выполнять резервное копирование определенных данных на устройстве DiskStation вручную или в соответствии с пользовательским расписанием. С помощью данной функции можно выполнить резервное копирование системной конфигурации, содержимого общих папок и данных пакетов в локальное, сетевое или облачное местоположение.

Создание задачи резервного копирования:

  1. Выберите Создать > Задача резервирования данных.
  2. Для завершения установки следуйте инструкциям мастера. Для получения дополнительной информации см. раздел Место назначения резервирования.

Управление задачами резервного копирования:

Выберите существующую задачу и нажмите одну из приведенных ниже кнопок.

  • Редактировать. Изменение местоположения, источника, расписания и других настроек выбранной задачи.
  • Удалить. Удаление выбранной задачи.
  • Отмена. Отмена выбранной выполняющейся задачи.
  • Резервировать сейчас. Немедленное резервное копирование выбранной задачи.

Ротация резервных копий

Ротация резервных копий — это механизм управления версиями резервных копий, который автоматически удаляет старые, ненужные версии, тем самым освобождая пространство для хранения данных. Эту функцию можно включить во вкладке Настройки задач или настроить ее при создании задачи резервного копирования.

Ротация версий:

  1. Выберите задачу резервного копирования, к которой необходимо применить ротацию, и нажмите Редактировать.
  2. Откройте вкладку Настройки задач и установите флажок для параметра Включить ротацию резервных копий.
  3. Здесь можно выбрать один из двух способов ротации:
    • С самой ранней версии. Это основной метод ротации. При активации условия ротации в первую очередь удаляется самая ранняя версия.
    • Smart Recycle. Это усовершенствованный метод ротации версий. При активации условия ротации система удалит версии, выбранные механизмом хранения. Механизм хранения выглядит следующим образом.
      • Ежечасные версии за последние 24 часа. Система хранит самую раннюю версию из создаваемых ежечасно.
      • Ежедневные версии от 1 прошедшего дня до 1 месяца. Система хранит самую раннюю версию из создаваемых ежедневно.
      • Еженедельная версия, созданная ранее 1 месяца назад. Система хранит самую раннюю версию из создаваемых еженедельно.
  4. Укажите максимальное количество версий. Когда версий станет больше этого количества, начнется ротация. В это количество не входят заблокированные версии.
  5. Если было настроено повторяющееся расписание резервного копирования, то в режиме предварительного просмотра будет отображаться способ сохранения версий резервных копий и самое раннее время восстановления на основе заданных настроек ротации.   
  6. Нажмите OK, чтобы сохранить изменения.

Примечания.

  • Заблокированные версии никогда не удаляются. Для получения дополнительной информации см. раздел Место назначения резервирования.
  • Функции ротации версий предусмотрены только для задач резервного копирования нескольких версий.
  • Ротация версий резервных копий не поддерживается в DSM 5.1 или более ранних версиях.
  • Ротация будет запущена сразу после изменения настройки или завершения задачи резервного копирования.
  • При расчете максимального количества сохраняемых версий заблокированные версии не учитываются.

Общие примечания:

  • Название задачи должно состоять из 1–32 символов Unicode и не должно содержать следующие символы:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • Максимальная длина полного пути к папке резервного копирования составляет 2048 символов.
  • При загрузке более 32 000 файлов или папок с помощью AFP в папку общего доступа, расположенную в файловой системе EXT3, все задачи резервного копирования, включая общую папку, не будут выполнены.
  • Зашифрованную папку можно копировать как подключенной, так и отключенной. Однако при восстановлении она всегда будет отключенной и не подключится автоматически при запуске. Вам будет необходимо ввести или импортировать ключ шифрования зашифрованной папки общего доступа для ее подключения.
  • Резервное копирование зашифрованной папки в другую зашифрованную папку не поддерживается.
  • Использование удаленных папок в качестве источника или места назначения резервного копирования не поддерживается.
  • Если местом резервного копирования является том, задачу сможет выполнить только владелец задачи. Для получения дополнительной информации см. раздел Место назначения резервирования.
  • Резервное копирование поддерживается для ряда приложений. Чтобы узнать, поддерживает ли определенный пакет резервное копирование, и просмотреть его содержимое и ограничения, см. справку для каждого приложения.
  • Устройство DiskStation не поддерживает резервное копирование приложений без внешнего устройства EXT3/EXT4 с возможностью записи.
  • Устройство DiskStation не поддерживает резервное копирование в том без внешнего устройства EXT3/EXT4 с возможностью записи.
  • DiskStation может выполнять резервное копирование только общих папок, созданных на внешних устройствах EXT3/EXT4.
  • Система не может выполнить резервное копирование приложений без тома с возможностью записи.

Резервное копирование настроек Windows ACL:

Настройки Windows ACL файлов на устройстве DiskStation могут копироваться в зависимости от файловой системы места назначения:

  • С помощью пунктов Место назначения локального резервирования, Место назначения сетевого резервирования и Место назначения публичного облачного резервирования можно копировать настройки Windows ACL независимо от целевой файловой системы.
  • Синхронизация общих папок позволяет синхронизировать настройки Windows ACL, если файловая система целевого устройства — EXT4/BTRFS.
