Восстановление

Выберите Резервное копирование и репликация > Восстановить, чтобы восстановить резервные данные.

Восстановление задачи резервного копирования:

  1. Выберите задачу резервного копирования и нажмите Восстановить. Если задача состоит в резервном копировании нескольких версий, выберите Восстановить > Вся задача.
  2. Вам будет предложено выбрать системную конфигурацию, различные версии резервных данных и др. Это будет зависеть от того, является ли это задачей резервного копирования данных, задачей резервного копирования LUN*, местом назначения резервного копирования одной версии (т. е. общей папкой) или местом назначения резервного копирования нескольких версий (т. е. томом).
  3. Для завершения установки следуйте инструкциям мастера.
  1. Выберите задачу резервного копирования и нажмите Восстановить.
  2. Вам будет предложено выбрать системную конфигурацию, различные версии резервных данных и др. Это зависит от типа задачи резервного копирования.
  3. Для завершения установки следуйте инструкциям мастера.

Примечание.

  • Восстановление поддерживается для ряда приложений. Чтобы узнать, поддерживает ли определенный пакет восстановление, и просмотреть его ограничения см. справку для каждого приложения.
  • При восстановлении пакета, который не установлен, система автоматически установит ту же или более новую версию приложения. Если версия установленного приложения ниже версии резервного копирования, система автоматически выполнит обновление до исходной скопированной версии.
  • Приложения, скопированные в DSM 5.1 или выше, не будут восстановлены в DSM 5.0 или более ранней версии.

Примечание.

  • После восстановления резервный LUN будет восстановлен в файловом LUN в выбранном томе.*
  • Восстановление поддерживается для ряда приложений. Чтобы узнать, поддерживает ли определенный пакет восстановление, и просмотреть его ограничения см. справку для каждого приложения.
  • При восстановлении пакета, который не установлен, система автоматически установит ту же или более новую версию приложения. Если версия установленного приложения ниже версии резервного копирования, система автоматически выполнит обновление до исходной скопированной версии.
  • Приложения, скопированные в DSM 5.1 или выше, не будут восстановлены в DSM 5.0 или более ранней версии.

Восстановление отдельного файла или папки для задачи резервного копирования с учетом нескольких версий:

  1. Запустите Обозреватель версий файлов. Это можно сделать любым из следующих способов:
    • Нажмите кнопку Восстановить и выберите в раскрывающемся меню Отдельные файлы/папки.
    • Выберите вкладку Место назначения резервирования и растяните выбранное место назначения. Нажмите значок Информация о версии резервной копии. В появившемся окне выберите резервную копию, которую нужно восстановить, и нажмите значок Браузер версий файлов.
  2. Версии нужного файла или папки можно просматривать с помощью временной шкалы или значка календаря в нижней части экрана.
  3. Выберите версию и нажмите кнопку Восстановить.
  4. Также можно скопировать файл или папку в другое место или скачать их, нажав кнопку Копировать в... или Загрузить.
  5. Кроме того, можно щелкнуть правой кнопкой мыши файл или папку и выбрать данные параметры в раскрывающемся меню.

Поиск конкретной версии:

  1. Нажмите стрелку в поле фильтрации в правом верхнем углу.
  2. В раскрывающемся меню можно фильтровать файлы по следующим параметрам:
    • Ключевое слово: Введите имя любого файла или папки. Вы можете использовать специальные символы [*] (один или несколько символов) и [?] (один произвольный символ) в ключевых словах. Например, [a*n] может означать [action], [attention] или [an].
    • Тип файла: чтобы сузить поиск, можно выбрать в раскрывающемся меню Любой, Любая папка или Любой файл.
    • Размер: выберите Любой или равен, более чем, менее чем, а затем укажите значение.
    • Диапазон дат: выберите Сегодня, Вчера, Последняя неделя, Последний месяц или Настроить и введите начальную и конечную дату для поиска версий в этом диапазоне.
  3. Чтобы применить фильтры, нажмите кнопку Фильтр.
  4. Чтобы сбросить поиск по фильтру, нажмите кнопку Сброс.

Примечание.

  • Функция браузера версий файлов предусмотрена только в задачах резервного копирования нескольких версий.
  • В DSM 5.1 и более ранних версиях обозреватель версий файлов имеет лишь некоторые функции.
  • Конфигурацию системы и приложения можно восстановить только посредством выбора параметров Восстановить > Вся задача.
  • Действия Копировать в... и Загрузить не распространяются на метаданные.

Восстановление задач резервного копирования из других мест назначения резервного копирования:

  1. Нажмите Восстановить из....
  2. Выберите один из следующих параметров для выбора задачи резервного копирования, а затем нажмите Далее:
    • Локальное восстановление. Выберите данный параметр для восстановления данных из локальной общей папки или внешнего диска.
    • Сетевое восстановление. Выберите данный параметр для восстановления данных с другого устройства DiskStation или с rsync-совместимого сервера.
    • Восстановление общей облачной службы. Выберите данный параметр для восстановления данных с публичного облачного сервера.
  3. Когда будет предложено, выберите источник, где находится задача резервного копирования, а затем выберите место назначения для восстановления данных.
  4. Следуйте инструкциям мастера.

Примечание.

  • Embedded DataStation может восстанавливать только общие папки на внешние устройства ext3/ext4.
  • Embedded DataStation не поддерживает восстановление приложений без внешнего устройства ext3/ext4.
