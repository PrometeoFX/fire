Обзор

На данной странице отображается базовая информация о кластере high-availability и серверах-участниках. Кроме того, на данной странице предоставляются инструменты для создания, удаления и выполнения обслуживания на кластерах high-availability и серверах-участниках.

Состояние кластера high-availability

В данном поле отображается общее состояние кластера high-availability, предупреждая о возможных ошибках. Ниже приведена таблица возможных условий, отображаемых в данном поле:

Состояние Описание
Отдельный Серверы не объединены в кластер high-availability.
Исправный Общее состояние кластера high-availability хорошее. Репликация данных с активного сервера на пассивный работает нормально.
Подготовка Серверы успешно объединены в кластер high-availability, но исходная репликация данных еще не завершена. Переключение функций в данный момент выполнить невозможно. Для получения дополнительной информации см. Переключение функций.
Загрузка Система загружает состояние кластера. Переключение функций выполнить невозможно до завершения загрузки состояния.
Внимание Ненормальное состояние кластера HA, но службы еще работают.
Опасность Активный и пассивный серверы неисправны. Кластер high-availability работает неисправно. Проверьте состояние всех объемов хранения (например, групп RAID, групп дисков, разделов и iSCSI LUN).

Предупреждающие сообщения

В следующей таблице приводятся предупреждающие сообщения, которые могут отображаться в случае возникновения ошибки в системе. При отображении более одного предупреждения, вы можете перемещаться между ними с помощью кнопок вперед/назад.

Внимание/Ошибка Описание
Ошибка контролируемой службы Произошла ошибка на одной или нескольких контролируемых службах.
Неисправность объема хранения Объем хранения (например, группа RAID, группа дисков, раздел, iSCSI LUN и твердотельный кэш-накопитель) на активном или пассивном сервере неисправен. Выполните проверку на странице Состояние системы хранения.
Повреждение объема хранения Объем хранения (например, группа RAID, группа дисков, раздел, iSCSI LUN и твердотельный кэш-накопитель) на активном или пассивном сервере поврежден. Выполните проверку на странице Состояние системы хранения.
Не удалось обнаружить пассивный сервер Данная ошибка может происходить из-за того, что пассивный сервер не в сети, или из-за невозможности подключения активного и пассивного сервера с помощью подключения Heartbeat. В данном примере переключение функций недоступно.
Невозможно выполнить репликацию данных на пассивный сервер Данная ошибка может возникать из-за сбоя раздела или диска. В данном примере переключение функций недоступно. Выполните проверку на странице Состояние системы хранения.
Пассивный сервер не выбран Если пассивный сервер отключен от кластера high-availability, это состояние будет отображено. В данном примере переключение функций недоступно. Чтобы повторно привязать пассивный сервер, нажмите раскрывающееся меню Обслуживание и выберите Привязать. При привязывании пассивного сервера потребуется выполнить полную репликацию данных.

Информация о кластере high-availability

Данное поле расположено под состоянием кластера high-availability и отображает основную информацию о кластере. Если кластер high-availability не существует, в данном поле будет отображаться только описание функции кластеров high-availability.

В данном поле содержится следующая информация:

  • Имя сервера кластера: Имя сервера используется кластером high-availability.
  • IP-адрес кластера: IP-адрес используется кластером high-availability. Независимо от того, какой сервер предоставляет услуги, пользователи могут подключаться к кластеру high-availability с помощью данного IP-адреса.
  • Время сборки: Дата создания кластера high-availability.

Функциональные кнопки кластера high-availability

Следующие две кнопки расположены под информацией о кластере high-availability:

  1. Создание кластера HA: Если кластер high-availability не существует, нажмите данную кнопку для объединения двух серверов и создания кластера high-availability. Для получения дополнительной информации о создании кластера high-availability см. Мастер Synology High Availability
  2. Управление: Данная кнопка доступна, если существует кластер high-availability. Включает следующие параметры:
    • Мастер управления: Запускает мастер, содержащий инструкции по различным процедурам управления.
    • Переключение функций: Переключение служб с активного сервера на пассивный сервер. Во время этого процесса активный сервер остановит предоставление услуг и станет пассивным. После этого пассивный сервер станет активным и начнет предоставлять услуги кластера high-availability. Данный параметр недоступен, если пассивный сервер имеет состояние ошибка или не выбран.
    • Обновление DSM: Открытие страницы Обновление DSM из Панели управления. Здесь можно обновить оба сервера.
    • Удалить кластер HA: Удаление кластера high-availability.
    • Освободить пассивный сервер: Освобождение пассивного сервера с кластера. Если сервер повторно добавлен в кластер high-availability после освобождения, потребуется выполнить полную репликацию данных. Поэтому рекомендуется использовать данный параметр только для замены существующего пассивного сервера новым.
    • Привязать пассивный сервер: Добавление пассивного сервера в кластер. Данный параметр доступен только если в кластере high-availability нет пассивного сервера.
    • Отключить кластер HA: Отключение активного и пассивного серверов.
    • Отключить активный сервер: Отключение активного сервера.
    • Отключить пассивный сервер: Отключение пассивного сервера.
    • Перезагрузить пассивный сервер: Перезагрузка пассивного сервера.
    • Отключить звуковой сигнал на пассивном сервере: Отсутствие звукового сигнала на пассивном сервере при возникновении ошибки. Пассивный сервер издаст звуковой сигнал, если произойдет другая ошибка.

Примечание.

  • Если после удаления кластера high-availability вам потребуется создать новый кластер, будет необходимо выполнить репликацию данных.
  • Если вы отключите или освободите пассивный сервер, службы будет невозможно переключить в случае возникновения ошибки на активном сервере.

Состояние сервера и репликации данных

В данном разделе приводятся изображения состояния активного и пассивного серверов, а также процесса репликации данных между серверами.

Изображение репликации данных указывает следующие возможные условия:

  • Репликация данных выполняется нормально: Обозначается потоком данных от активного сервера к пассивному.
  • Репликация данных не выполняется: Обозначается отсутствием изображения потока данных.

Информация о сервере

Данный раздел расположен под изображениями сервера. В данном разделе отображается информация об активном и пассивном серверах. Если пассивный сервер не выбран, в данном разделе будет отображаться состояние Недоступно.

  • Имя сервера: Имя сервера Synology.
  • Название модели: Модель сервера Synology.
  • Серийный номер: Серийный номер сервера Synology.
  • Состояние вентилятора: Состояния вентилятора сервера. Если вентилятор неисправен, отобразится предупреждение. В случае неисправности обратитесь за помощью к продавцу.
  • Температура: Температура сервера Synology.
  • Состояние питания: Отображает состояние питания на сервере. Если модель сервера оснащена двумя источниками питания, отобразится предупреждение при неисправности одного из них.
