Redes divididas

Quando um conjunto de high-availability estiver funcionando normalmente, apenas um dos servidores membros assume o papel de servidor ativo. Nesse caso, o servidor passivo detecta a presença do servidor ativo por meio da conexão Heartbeat e da conexão de dados.

Se todas as conexões Heartbeat e de dados forem perdidas, os dois servidores poderão tentar assumir o papel de servidor ativo. Essa situação é chamada de erro de "redes separadas". Nesse caso, as conexões para o endereço IP atribuído ao conjunto de high-availability serão redirecionadas para um dos dois servidores, o que resultará em erros de serviço.

Caso ocorra um erro de redes separadas:

  1. As conexões para o endereço IP do conjunto serão redirecionadas para qualquer um dos servidores, o que resultará em erros de serviço. Você enfrentará dificuldade ao se conectar a qualquer servidor.
  2. O sistema detectará o erro de redes divididas e removerá o conjunto de high-availability, desvinculando os dois servidores.
  3. Quando os dois servidores voltares para o status Independente você conseguirá entrar no DiskStation Manager (DSM). Nesse momento, os dados acumulados antes da remoção do conjunto de high-availability permanecerão nos dois servidores. Após identificar qual servidor contém os dados mais atualizados, você pode vincular os servidores novamente para criar um conjunto de high-availability novamente.
