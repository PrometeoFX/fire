Serviço de arquivos do Windows

O serviço de arquivos do Windows permite que clientes Windows, Mac e Linux com suporte a SMB/CIFS acessem dados armazenados no DiskStation. Se você desejar procurar arquivos no DiskStation através do Windows Explorer ou montar pastas compartilhadas como unidades de rede, ative essa opção em Painel de controle > Serviços de arquivos > Win/Mac/NFS.

Para ativar o serviço de arquivos Windows:

  1. Marque Ativar serviço de arquivos Windows.
  2. Clique em Aplicar.

Grupo de trabalho

Esta opção permite que você designe um sistema como membro de um grupo de trabalho específico para fins de gerenciamento de rede.

Nota:

  • O nome de um grupo de trabalho pode conter de 1 a 15 caracteres. O nome não pode ter os seguintes caracteres: [ ] ; : " < > * + = \ / | ? ,
  • Se você estiver usando um Mac que executa o OS X 10.9 e seu DiskStation estiver ingressado em um domínio do Windows, poderá entrar adicionando localhost\ antes do seu nome de usuário local do DiskStation para acessar arquivos armazenados no DiskStation através do serviço de arquivos do Windows.

Desabilitar acesso a versões anteriores

No Windows Explorer, os usuários podem acessar instantâneos de pastas ou arquivos clicando na guia Versões anteriores dentro de uma pasta compartilhada, o que permite a restauração ou cópia de dados sem a ajuda do administrador. Se você escolher Desabilitar acesso a versões anteriores, a página “Versões anteriores” não exibirá nenhuma informação.

Observação:

  • Os usuários têm permissão de acessar apenas uma versão anterior se, no momento em que o instantâneo correspondente à versão anterior foi tirado, suas contas de usuários foram criadas ou receberam permissão de leitura da pasta compartilhada à qual corresponde o instantâneo tirado.

Ativação do registro de transferência

Marcar essa opção ativará o registro das operações de acesso a arquivos, incluindo criação, exclusão, leitura, gravação, transferência e renomeação de arquivos ou pastas através do protocolo CIFS/SMB, também conhecido como protocolo de transferência de arquivos do Windows.

Configurações avançadas

Clique no botão Configurações avançadas para ver as opções abaixo.

  • Servidor WINS: O Serviço de cadastramento na Internet do Microsoft Windows (WINS) é um serviço de mapeamento de nome NetBIOS para endereço de IP. Os usuários do Windows localizarão o sistema em redes TCP/IP com mais facilidade se o sistema tiver sido configurado para ser registrado em um servidor WINS.
  • Protocolo máximo para serviço de arquivos do Windows: Dependendo da sua configuração de rede, você pode definir o mais alto nível de protocolo que será suportado pelo seu DiskStation.
    • Habilitar SMB 2: O SMB (Server Message Block) 2 tem suporte desde o Windows Vista e é uma versão aprimorada do SMB. O SMB 2 adiciona a capacidade de incorporar várias ações SMB em uma única solicitação a fim de reduzir o número de pacotes de rede e melhorar o desempenho.
    • Habilitar SMB 2 com MTU grande: A unidade máxima de transmissão (MTU) refere-se à maior unidade de dados que pode ser transmitida através de um protocolo de comunicação. Para aproveitar interfaces de rede mais rápidas, como Ethernet de 1 ou 10 gigabits, a Large MTU aumenta a unidade de transmissão máxima para 1 MB. A ativação da Large MTU aumenta a velocidade e a eficiência da transferência de arquivos grandes e, ao mesmo tempo, reduz o número de pacotes que precisam ser processados.
    • Habilitar SMB 3: O SMB 3 é suportado desde o Windows 8 e Windows Server 2012. Ele é uma versão aprimorada do SMB 2. O SMB 3 suporta a transmissão por criptografia de arquivo baseada em AES, aumentando assim a segurança da transferência de arquivos ponto a ponto.

    Nota:

    • O SMB 2 tem suporte no Windows Vista, no Windows Server 2008 R2, no Windows 7 e em versões mais recentes.
    • A Large MTU tem suporte no Windows Server 2008 R2, no Windows 7 e em versões mais recentes.
    • Para tornar eficiente o Large MTU em seu computador com Windows, será necessário modificar as configurações de registro do computador. Para obter mais detalhes, consulte o suporte técnico da Microsoft (por exemplo, este artigo).
  • Modo de criptografia de transporte: Quando o SMB 3 é habilitado, o Serviço de arquivos do Windows adicionará criptografia de transporte para fortalecer a segurança da transmissão de arquivos.
    • Desabilitar: Nenhuma criptografia de transporte será aplicada.
    • Auto: A criptografia de transporte será aplicada apenas a clientes que suportam esse recurso.
    • Forçar: A criptografia de transporte será aplicada sempre. Isso fará com que os clientes que não suportam a criptografia de transporte não consigam usar o serviço de arquivos do Windows.

    • Nota:

    • A criptografia de transporte é suportada no Windows 8, Windows Server 2012 e em versões mais recentes.
    • O login anônimo no serviço de arquivos do Windows não é suportado quando a criptografia de transporte está habilitada.
    • Quando a criptografia de transporte for habilitada, deixe a opção Desabilitar Opportunistic Locking desmarcada para evitar atingir os tempos limite do aplicativo.
  • Desabilitar Opportunistic Locking: Marcar esta opção desabilita o "Bloqueio oportunista" do CIFS, resultando em melhor desempenho quando aplicativos de bancos de dados acessarem arquivos de bancos de dados na pasta compartilhada do dispositivo DiskStation pela rede.
  • Ativar Local Master Browser: Quando existem vários computadores com Windows na mesma subrede, eles negociarão e elegerão um computador como o "local master browser". O local master browser mantém listas de outros computadores dentro da subrede e seus recursos compartilhados e compartilha essas listas com outros computadores. Essa opção permite que o DiskStation assuma o papel de local master browser.

    • Nota:

    A habilitação desta função desabilita a hibernação do disco rígido e ativa a conta do convidado sem uma senha.

  • Habilitar o módulo VFS DirSort: Classifica os nomes de pasta e lista-os em ordem alfabética. Ativar esta opção afeta o desempenho do sistema.
  • Arquivos de veto: Oculta os arquivos de acordo com os critérios especificados. Os critérios podem incluir curingas (*) e entradas múltiplas devem ser separadas por barras (/). Por exemplo: /abc*/*.txt/*.conf/. Ativar esta opção afeta o desempenho do sistema.
  • Permitir links simbólicos nas pastas compartilhadas: Ativar essa opção permite que links simbólicos apontem para arquivos ou pastas na mesma pasta compartilhada. Eles podem ser criados usando o terminal de DSM.
  • Permitir links simbólicos entre pastas compartilhadas: Ativar essa opção permite que os links simbólicos apontem para arquivos ou pastas em diferentes pastas compartilhadas. Eles podem ser criados usando o terminal de DSM. Observe que ativar esta opção pode permitir que usuários sem privilégios de acesso visualizem e acessem pastas compartilhadas que são alvo dos links simbólicos.
  • Habilitar o módulo VFS MSDFS: Permite que links simbólicos apontem para arquivos ou pastas localizadas em outros servidores SMB. Com versões mais antigas do Windows, como o Windows XP, o cliente e o DiskStation precisam participar de um domínio do Windows antes de ativar essa opção. Para saber mais sobre como criar os links MSDFS, consulte aqui. Observe que o cache do dispositivo cliente deve ser limpo após desabilitar esta opção, ou o desempenho normal poderá ser afetado.
  • Desabilitar múltiplas conexões do mesmo endereço IP: Quando essa opção estiver ativada, o DiskStation só permite uma conexão por endereço IP de clientes usando o Serviço de arquivos do Windows. Quando vários clientes com o mesmo endereço IP usarem o Windows File Service, o DiskStation irá manter o mais recente e derrubar as conexões mais antigas. Em ambientes NAT, o cliente poderá usar um único endereço de IP, caso em que se ativar essa opção isso poderá afetar o desempenho normal.
  • Coletar registros de depuração: Quando essa opção for ativada, registros detalhados serão armazenados para fins de depuração. Ativar esta opção afeta o desempenho do sistema.
  • Aplique as permissões UNIX padrão: Habilite essa opção para aplicar as permissões UNIX padrão ao fazer upload ou criar arquivos e pastas. A permissão UNIX será 744 para arquivos e 755 para pastas. Quando essa opção é desativada, a permissão UNIX é 777 para arquivos e pastas.

    • Nota:

    • Habilitar essa opção pode causar problemas de permissão inconsistente entre os diferentes protocolos. Para evitar inconsistências, sugerimos deixar essa opção desabilitada.

    Nota:

    • Para pastas compartilhadas com a ACL do Windows habilitada (todos os compartilhamentos, exceto "photo" e compartilhamentos em unidades externas), execute o comando chmod no cliente Linux ou FTP para alterar os tipos de permissão de arquivos e pastas da ACL do Windows para UNIX.
    • Habilitar essa opção pode causar problemas de permissão inconsistente entre os diferentes protocolos. Para evitar inconsistências, sugerimos deixar essa opção desabilitada.
  • Não reservar espaço em disco ao criar arquivos: Habilite essa opção para não permitir que espaço em disco seja reservado ao se criar arquivos no seu DiskStation .
  • Apagar cache SMB: Clique neste botão para excluir os seguintes arquivos do cache SMB, para fins de solução de problemas:
    • /var/cache/samba/netsamlogon_cache.tdb
    • /var/lib/samba/winbindd_cache.tdb
    • /var/lib/samba/share_info.tdb
    • /var/lib/samba/registry.tdb
    • /run/samba/connections.tdb
    • /run/samba/gencacche.tdb
    • /run/samba/gencache_notran.tdb
