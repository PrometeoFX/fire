Como fazer backup de dados em meu PC/Mac usando o Cloud Station

Visão geral

O Cloud Station não é apenas um sofisticado software de sincronização multiplataforma, os usuários também podem usá-lo como uma maneira mais inteligente de fazer backup de seus computadores pessoais. Esses são alguns benefícios de usar o Cloud Station como auxiliar de backup. Em primeiro lugar, a replicação em tempo real fornece um RPO melhor, um período mais curto em que os dados podem se perder (irrecuperáveis) de seu PC devido a um grande incidente. Ele não apenas é instantâneo como também o ajudará a economizar largura de banda drasticamente em comparação com o backup tradicional que usa grandes quantidades de largura de banda em um determinado momento. Você pode filtrar facilmente os arquivos dentro da mesma pasta que você não quer fazer backup, configurando o tamanho ou o tipo do arquivo em seu cliente PC. O Cloud Station também pode manter até 32 versões de histórico de um único arquivo que o Cloud Station permite que você restaure a um cliente PC do lado do servidor (NAS).

1. Configuração do Cloud Station no Synology NAS

  1. Entre no DSM usando uma conta pertencente ao administrator.
  2. Abra o Cloud Station.
  3. Se você não habilitou o recurso de página de início do usuário, o sistema pedirá que você o faça. Clique em OK.
  4. Você será redirecionado para a página de configurações do usuário. Habilite as páginas iniciais do usuário e clique em Aplicar.
  5. Agora volte para o Cloud Station e o sistema solicitará para habilitá-lo. Clique em Sim.
    6. Observação: O QuickConnect é um serviço de retransmissão que permite que você se conecte facilmente ao seu Synology NAS de fora da rede local. Se quiser sincronizar seu arquivo longe de casa ou do escritório, recomendamos que você configure o QuickConnect indo até Painel de controle > QuickConnect.
  6. Vá para Privilégios e especifique quais usuários poderão usar o Cloud Station. Clique em Salvar.
  7. Configure uma pasta compartilhada em seu Synology NAS para fins de backup. Para fazer isso, vá para Painel de controle > Pasta compartilhada e clique em Criar.
  8. Nomeie sua pasta e preencha os detalhes da forma correspondente. Clique em OK. Por exemplo, nós nomearemos a pasta como Backup.
  9. Na guia Permissões, especifique quais usuários poderão acessar a pasta. Por exemplo, nós daremos ao admin permissões de leitura/gravação para a pasta. Clique em OK.
  10. Vá para Cloud Station > Configurações > Compartilhamento e selecione a pasta Backup e clique em Habilitar.

2. Configuração do Cloud Station em seu PC Windows

Siga as etapas para configurar o Cloud Station em seu Mac ou PC

2.1 Para PC

  1. Vá para Cloud Station > Visão geral e clique no botão Download.
  2. Clique no botão para baixar o Cloud Station para PC Windows. (Você também pode baixar o Cloud Station de Centro de Download Synology.)
  3. Inicie o instalador em seu computador e siga as instruções para instalar o Cloud Station.
  4. Após a instalação, execute o Cloud Station em seu computador. Clique em Iniciar Agora.
  5. Digite o endereço do Synology NAS executando o Cloud Station (ou QuickConnect ID), nome de usuário e senha. (Você também pode clicar no ícone à direita e o Cloud Station pesquisará automaticamente outro Synology NAS dentro de sua LAN). Clique em Avançar.
    6. Observação: Verifique se a conta inserida tem permissões concedidas em Cloud Station > Privilégios no Synology NAS.
  6. Selecione como deseja configurar sua tarefa (aqui nós selecionaremos Configuração avançada pois estamos configurando uma pasta especificamente para backup). Clique em Avançar.
  7. Selecione uma pasta no seu computador para manter sincronizada com a pasta Backup que você criou anteriormente em seu Synology NAS. Clique em Avançar.
  8. Clique em Concluído para completar a configuração.

Para criar outra tarefa:

  1. Vá para Cloud Station > Configurações e habilite uma pasta compartilhada para essa nova tarefa. Meu exemplo é foto.​ Certifique-se de ter privilégios de leitura/gravação para foto.
  2. De volta ao Cloud Station em seu PC, clique em Criar.
  3. ​Selecione um Synology NAS para sua nova tarefa Aqui selecionaremos Synology NAS conectado pois estamos criando outra tarefa para o mesmo Synology NAS. Clique em Avançar.​
  4. Clique em Configuração avançada e siga o restante do assistente para continuar a configuração.

3. Recuperar dados do Cloud Station

Se você remover ou trocar um arquivo acidentalmente, o Cloud Station irá permitir recuperá-lo facilmente.

4.1 Baixar uma versão anterior de um arquivo sincronizado no PC

  1. Clique no ícone Cloud Station na bandeja do seu sistema e no ícone da pasta.
  2. Clique com o botão direito no arquivo selecionado e escolha Synology Cloud Station > Procurar versões anteriores.
  3. Localize a versão que deseja baixar e clique no ícone de download.
  4. Insira o nome do arquivo e escolha o destino para salvar o arquivo.

4.1 Baixar uma versão anterior de um arquivo sincronizado no Mac

  1. Clique no ícone Cloud Station na bandeja do seu sistema e no ícone da pasta.
  2. Clique com o botão direito no arquivo selecionado e escolha Synology Cloud Station > Procurar versões anteriores.
  3. Localize a versão que deseja baixar e clique no ícone de download.
  4. Insira o nome do arquivo e escolha o destino para salvar o arquivo.

4.3 Baixar ou restaurar uma versão anterior de um arquivo sincronizado no Cloud Station

  1. Vá para Cloud Station > Histórico de Versões e selecione a pasta Backup criada anteriormente.
  2. Selecione o arquivo que deseja recuperar e clique em Ação > Procurar versões anteriores.
  3. Selecione a versão do arquivo que deseja baixar e clique em Download. Ou, se você tem certeza de que quer a versão selecionada restaurada, clique em Restaurar e ela substituirá a versão atual.

4. Observações importantes

  1. Sempre mantenha versões históricas no caso de um arquivo ser removido ou substituído por acidente.
  2. Em Configurações Globais, você pode configurar se um arquivo será buscado novamente em seu NAS ou excluído em seu NAS quando você excluir um arquivo do PC.
