iSCSI LUN*

iSCSI é um serviço de rede de armazenamento que facilita a transferência de dados em redes locais ou longas distâncias. Ao mapear iSCSI LUNs para iSCSI Targets, os clientes podem acessar os espaços de armazenamento como se fossem discos locais. Um iSCSI LUN (número da unidade lógica) pode ser mapeado para um ou vários iSCSI Targets para fornecer serviços de armazenamento iSCSI. Vá para Gerenciador de armazenamento > iSCSI LUN para gerenciar os iSCSI LUNs.

Nota:

O número de iSCSI LUNs suportados varia nos modelos do DiskStation. Para obter as especificações detalhadas do produto, consulte www.synology.com.

DiskStation atualmente suporta os seguintes tipos de iSCSI LUNs:

  • iSCSI LUN (Arquivos Comuns): o iSCSI LUN é criado ao atribuir uma parte do espaço do volume para seu uso. Você pode expandir seu tamanho posteriormente quando isso for necessário. Este tipo de iSCSI LUN fornece flexibilidade do gerenciamento dinâmico de capacidade com o Thin Provisioning e o suporte de aceleração de hardware VMware VAAI, suporte ao Windows ODX, instantâneo de LUN e clone de LUN.
  • iSCSI LUN (Nível de bloqueio) - LUN único no RAID: o iSCSI LUN é criado usando todo o espaço de um volume. Você pode repará-lo ou expandi-lo, ou alterar seu tipo RAID, da mesma forma que usando um volume. Consulte a página Volume para obter mais informações.
  • iSCSI LUN (Nível de bloqueio) - LUNs Múltiplo no RAID: o iSCSI LUN é criado designando-se um espaço específico de um Grupo de discos. Você pode expandir o iSCSI LUN posteriormente se houver espaço disponível no Grupo de discos. Consulte a página Grupo de discos para obter mais informações.
  • iSCSI LUN (Nível de bloqueio): Esse tipo de iSCSI LUN é criado no RAID Group e fornece a flexibilidade do gerenciamento dinâmico de capacidade com desempenho de acesso otimizado. Você pode expandir esse tipo de iSCSI LUN posteriormente usando o espaço não alocado do RAID Group. Consulte a página RAID Group para obter mais informações.

Nota:

iSCSI LUNs com recursos LUN avançados estão disponíveis apenas em sistemas de arquivo ext4/btrfs.

Para criar um iSCSI LUN:

  1. Clique em Criar.
  2. Selecione o tipo de iSCSI LUN que deseja criar.
  3. Siga o assistente para concluir o processo.

Nota:

Os recursos LUN avançados podem ser habilitados apenas durante a criação do iSCSI LUN inicial. Essa configuração não pode ser alterada posteriormente.

O tamanho da unidade de alocação LUN avançada pode ser otimizado para determinadas características do servidor de aplicativos, como o Microsoft Exchange ou o SQL Server, ou servidores de virtualização e tecnologias de aceleração de armazenamento, como o Microsoft Windows ODX ou VMware VAAI, fazendo um dos seguintes procedimentos:

  • Defina o tamanho da unidade de alocação durante a criação do primeiro LUN avançado de um volume.
  • Defina o tamanho da unidade de alocação em Configurar antes de criar o primeiro LUN avançado de um volume.

Recomendamos que você especifique o tamanho da unidade de alocação para seu aplicativo ao utilizar uma conexão direta de seu servidor de aplicativos ou métodos de trabalho RDM (pass-through) com o iSCSI LUNs avançados no DiskStation. Para tirar proveito da tecnologia de LUN avançado da Synology para clonagem para Windows ODX, sugerimos que você formate o volume NTFS com o tamanho da unidade de alocação igual (ou maior que) a que você especificou no DSM.

Nota:

A unidade de alocação LUN avançada de cada volume pode ser definida apenas uma vez.

Para remover um iSCSI LUN:

Ao remover um iSCSI LUN, todos os dados no iSCSI LUN serão excluídos. As conexões entre o iSCSI LUN e qualquer iSCSI Targets mapeado também será perdida.

  1. Selecione o iSCSI LUN que deseja remover.
  2. Clique no botão Remover e siga o assistente para concluir o processo.

Nota:

Excluir os VAAI LUNs do DiskStation não libera espaço em disco imediatamente. Os dados são transferidos para uma pasta reservada do sistema chamada @EP_trash e excluídos parte por parte. Por exemplo, se um VAAI LUN de 100 GB for excluído do DiskStation, podem ser necessários aproximadamente 15-20 minutos para liberar 100 GB de espaço no volume do disco.

Para forçar a montagem do iSCSI LUN (Nível de bloqueio):

Se o cache em iSCSI LUN (Nível de bloqueio) estiver ausente, o sistema não montará o iSCSI LUN (Nível de bloqueio) durante a inicialização. Se o cache não for recuperável e você quiser acessar o iSCSI LUN (Nível de bloqueio), clique em Forçar montagem.

Nota:

Essa função está disponível apenas em modelos específicos.

Para editar um iSCSI LUN:

  1. Selecione o iSCSI LUN que deseja editar.
  2. Clique em Editar.
  3. Edite as propriedades desejadas.
  4. Clique em OK para salvar as configurações.

Nota:

Para evitar a perda de dados quando editar a capacidade do iSCSI LUN, você só poderá fazê-lo até o tamanho atual.

Para tirar um instantâneo de iSCSI LUN:

Um instantâneo de iSCSI LUN é um ponto no tempo, cópia somente de leitura de um iSCSI LUN.

  1. Selecione o iSCSI LUN do qual deseja tirar um instantâneo.
  2. Clique em Instantâneo e escolha Tirar um instantâneo.
  3. Digite uma descrição para este instantâneo.
  4. Escolha um dos seguintes tipos de instantâneos de acordo com suas necessidades:
    • Aplicativo consistente: Este tipo de instantâneo só pode ser tomado quando o Synology Snapshot Manager estiver instalado no seu servidor Windows ou VMware vCenter Server, dependendo do ambiente operacional para se comunicar com o DSM. Quando os instantâneos consistentes com o aplicativo forem acionados no DSM, os aplicativos serão notificados e liberarão todos os dados da memória para o LUN no qual o instantâneo foi tirado para garantir a consistência dos dados.
    • Falha consistente: se o Synology Snapshot Manager não estiver instalado em nenhum dos seus clientes, só será possível capturar instantâneos consistentes com paralisação de LUNs. Quando eles forem acionados, os aplicativos não serão notificados para liberar os dados da memória para o LUN e, portanto, não garantirão a consistência dos dados.
  5. Clique em OK.

Para restaurar um iSCSI LUN:

Selecione um instantâneo pontual para restaurar os dados em um iSCSI LUN.

  1. Selecione o iSCSI LUN que deseja restaurar.
  2. Clique em Instantâneo e selecione Gerenciador de instantâneo.
  3. Selecione um instantâneo que deseja restaurar e clique em Restaurar.

Para clonar um iSCSI LUN ou instantâneo:

Faça uma cópia gravável do iSCSI LUN ou de um instantâneo do iSCSI LUN.

  • Selecione o iSCSI LUN na lista e clique em Clonar.
  • Selecione um instantâneo do iSCSI LUN no Gerenciador de instantâneo e clique em Clonar.

Nota:

  • Se houver pelo menos uma sessão conectada de iSCSI associada ao LUN e estiver sendo usada pelo Windows Server ou VMware ESXi Server, recomendamos instalar o Synology Snapshot Manager para evitar possíveis inconsistências de dados, caso contrário, você deverá interromper todos os aplicativos e fazer logoff das sessões dos iniciadores antes de prosseguir.
  • As funções clonar e restaurar e instantâneo de LUN estão disponíveis apenas em iSCSI LUNs com recursos LUN avançados.
  • Se você tirou um instantâneo ou criou um clone e o status mudou para Anormal/Paralisado, exclua esse instantâneo/clone e crie outro.
  • Todas as alterações feitas a um LUN após um instantâneo ter sido criado serão perdidas quando você restaurar o LUN para esse instantâneo.
  • Se o status do iSCSI LUN mudar para Indisponível, significa que ocorreram alguns erros relacionados aos dados durante o acesso ao iSCSI LUN. Para evitar perdas graves de dados do sistema, esse iSCSI LUN ficará temporariamente inacessível. Entre em contato com a Synology para obter suporte assim que possível.

Glossário:

  • Thin Provisioning: Thin Provisioning é um método de otimização do armazenamento pela alocação do espaço de armazenamento de forma dinâmica e sob demanda. Esta configuração está disponível apenas para iSCSI LUN (arquivos regulares).
  • VMware VAAI: as APIs VMware vStorage para Integração de matriz (VAAI) pode desfazer-se do trabalho das operações padrão das matrizes de armazenamento virtual para o dispositivo iSCSI no ambiente VMware vSphere e otimizar o desempenho do armazenamento.
  • Windows ODX: uma nova tecnologia de transferência de dados desenvolvida pelo Windows no Windows Server 2012 e 8 chamada Offload Data Transfer (ODX). O ODX aprimora o desempenho de transferência descarregando alguma carga de trabalho para a matriz de armazenamento.
    Se a fonte e o destino dos dados estiverem em LUNs localizados no mesmo volume, a clonagem será processada pela tecnologia Synology Advanced LUN, que poupa até 99,9% do consumo de espaço de armazenamento e acelera a clonagem ao mesmo tempo.
  • Mapeamento de iSCSI Target: mapeia o iSCSI LUN para um ou mais iSCSI Targets.
  • Synology Snapshot Manager: o Synology Snapshot Manager é um plugin de software que permite tirar instantâneos consistentes de aplicativo diretamente do DiskStation Manager (DSM).
    • No ambiente do VMware, você precisará instalar o Synology Snapshot Manager no servidor do Windows, onde o VMware vCenter Server está instalado. Quando um instantâneo é ativado no DSM, o VMware vCenter Server será notificado e liberará todos os dados da memória para o LUN no qual o instantâneo foi tirado para garantir a consistência dos dados. Depois que o instantâneo estiver concluído, o VMware vCenter Server retomará a operação normal de E/S de armazenamento de dados do VMware. O Synology Snapshot Manager para VMware vCenter Server é compatível com o vSphere 5.1 ou versões mais novas.
    • No ambiente Windows, você precisará instalar o Synology Snapshot Manager em seu servidor do Windows. Quando um instantâneo é ativado no DSM, o Synology Snapshot Manager aproveitará a tecnologia Volume Shadow Copy Service (VSS) da Microsoft para produzir cópias de dados pontuais consistentes. O Synology Snapshot Manager for Windows é compatível com o Windows Server 2008 R2, 2012 e 2012 R2.
    • O Synology Snapshot Manager pode ser baixado gratuitamente no site www.synology.com.
