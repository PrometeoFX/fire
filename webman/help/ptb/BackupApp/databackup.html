Backup de dados

Na guia Backup em Backup e replicação, é possível fazer backup de dados específicos no DiskStation manualmente ou de acordo com uma programação personalizada. Com esse recurso, as configurações do sistema, o conteúdo das pastas compartilhadas e os dados do pacote podem ser copiados para um destino de backup local, em rede ou na nuvem.

Para criar uma tarefa de backup:

  1. Selecione Criar > Tarefa de backup de dados.
  2. Siga o assistente para concluir a configuração. Visite Destino do backup para obter mais informações.

Para gerenciar as tarefas de backup:

Selecione uma tarefa existente e clique em um dos botões abaixo:

  • Editar: Modificar destino, origem, agendamento e outras configurações da tarefa selecionada.
  • Excluir: Excluir a tarefa selecionada.
  • Cancelar: Cancelar a tarefa selecionada que está sendo executada no momento.
  • Fazer backup agora: Executa imediatamente o backup da tarefa selecionada.

Rotação do backup

A rotação de backup é um mecanismo de gerenciamento de versão. Ele ajuda a gerenciar as versões de backup excluindo automaticamente as versões mais antigas e desnecessárias e liberando espaço de armazenamento. Você pode habilitar essa função em Configurações de tarefa ou configurá-la quando você criar uma tarefa de backup.

Para rotacionar as versões:

  1. Selecione a tarefa de backup para a qual deseja aplicar a rotação de versão e clique em Editar.
  2. Vá para a guia Configurações de tarefa e marque Habilitar rotação de backup.
  3. Aqui você pode escolher um ou dois métodos de rotação:
    • Da versão mais antigas: Esse é um método básico de rotação de versão. Quando a condição de rotação é disparada, a versão mais antiga será removida primeiro.
    • Smart Recycle: Esse é um método inteligente de rotação de versão. Quando a condição de rotação é disparada, o sistema remove as versões selecionadas pelo mecanismo de retenção. O mecanismo de retenção é o seguinte:
      • Versões horárias das últimas 24 horas: O sistema mantém a versão mais recente que foi criada em cada hora.
      • Versões diárias de 1 dia passado a 1 mês: O sistema mantém a versão mais recente que foi criada em cada dia.
      • Versão semanal com mais de 1 mês: O sistema mantém a versão mais recente que foi criada em cada semana.
  4. Especifique o número máximo de versões. Quando o número de versões ultrapassar esse número, a rotação de versões começará. O número não inclui as versões bloqueadas.
  5. Se você configurou uma programação de backup repetida, uma visualização da estimativa de como suas versões de backup serão mantidas e o tempo do ponto de recuperação mais recente baseado nas configurações de rotação serão exibidos.   
  6. Clique em OK para salvar suas configurações.

Notas:

  • As versões bloqueadas nunca serão removidas. Visite Destino do backup para obter mais informações.
  • Apenas as tarefas de backup multiversão fornecem funções de rotação de versão.
  • A rotação de versão não é suportada no DSM 5.1 ou inferior.
  • A rotação será disparada logo após a configuração ser alterada ou a tarefa de backup concluída.
  • As versões bloqueadas são excluídas ao se calcular o número máximo de versões.

Observações gerais:

  • Os nomes das tarefas podem incluir de 1 a 32 caracteres Unicode, mas não podem conter os seguintes caracteres:
    ! " # $ % & ' ( ) * + , / : ; < = > ? @ [ ] \ ^ ` { } | ~
  • O comprimento máximo do caminho completo da pasta para backup é de 2048 caracteres.
  • Se mais de 32.000 arquivos ou pastas forem carregados para uma pasta compartilhada EXT3 via AFP, todas as tarefas de backup que incluem a pasta compartilhada falharão.
  • Pode-se fazer backup de uma pasta criptografada montada ou desmontada. Entretanto, ela será sempre restaurada como desmontada e não será montada automaticamente na inicialização. O sistema solicitará que você digite ou importe a chave de criptografia da pasta compartilhada criptografada para montá-la.
  • Não é suportado fazer backup de uma pasta criptografada para outra pasta criptografada.
  • Não é suportado usar pastas remotas como fonte ou destino de backup.
  • Se o destino do backup for um volume, somente o proprietário da tarefa poderá executar a tarefa de backup. Visite Destino do backup para obter mais informações.
  • O backup de determinados aplicativos é suportado. Para saber se um pacote específico suporta o backup e conhecer o conteúdo do backup e suas limitações, consulte a ajuda de cada aplicativo.
  • O DiskStation não suporta o backup de aplicativos sem um dispositivo externo EXT3/EXT4 gravável.
  • O DiskStation não suporta o backup para um volume sem um dispositivo externo EXT3/EXT4 gravável.
  • O DiskStation só pode fazer backup de pastas compartilhadas criadas em dispositivos externos EXT3/EXT4.
  • O sistema não pode fazer backup de aplicativos sem um volume gravável.

Para fazer backup de configurações de ACL do Windows:

Também é possível fazer o backup das configurações de ACL do Windows dos arquivos no DiskStation, dependendo dos sistemas de arquivos do destino:

  • O Destino de backup local, o Destino de backup de rede e o Destino de backup de nuvem pública podem fazer backup das configurações de ACL do Windows, independentemente do sistema de arquivos do destino.
  • A Sincronização de pasta compartilhada pode fazer backup das configurações de ACL do Windows quando o sistema de arquivos do destino for EXT4/BTRFS.
